# [GESP202409 六级] 算法学习

## 题目描述

小杨计划学习 $m$ 种算法，为此他找了 $n$ 道题来帮助学习，每道题最多学习一次。

小杨对 $m$ 种算法的初始掌握程度均为 $0$。第 $i$ 道题有知识点 $a_i$，学习它可将第 $a_i$ 种算法的掌握程度提高 $b_i$。目标是每种算法的掌握程度均至少达到 $k$。

小杨认为连续学习两道相同知识点的题是不好的，需要避免。求最少需要学习多少道题才能在完成目标的同时避免连续学相同知识点的题。如果不存在方案，输出 $-1$。

## 输入格式

第一行三个正整数 $m,n,k$。

第二行 $n$ 个正整数 $a_1,a_2,\dots,a_n$。

第三行 $n$ 个正整数 $b_1,b_2,\dots,b_n$。

## 输出格式

一行一个整数，表示最少需要学习的题目数，无解输出 $-1$。

## 样例

### 样例 1

```
输入：
3 5 10
1 1 2 3 3
9 1 10 10 1

输出：
4
```

**解释**：学习顺序为第 $1,3,4,2$ 道题（知识点 $1,2,3,1$），掌握程度分别为 $(9+1,10,10)=(10,10,10)$。

### 样例 2

```
输入：
2 4 10
1 1 1 2
1 2 7 10

输出：
-1
```

## 提示

| 子任务 | 占比 | $m$ | $n$ | $b_i$ | $k$ |
| :-: | :-: | :-: | :-: | :-: | :-: |
| $1$ | $30\%$ | $2$ | $\le 9$ | $\le 10$ | $\le 10$ |
| $2$ | $30\%$ | $\le 9$ | $\le 9$ | $\le 10$ | $\le 10$ |
| $3$ | $40\%$ | $\le 10^5$ | $\le 10^5$ | $\le 10^5$ | $\le 10^5$ |

对于全部数据，$1\le m,n,b_i,k\le 10^5$，$1\le a_i\le m$。

## 解题思路

### 问题分析

每种算法需要至少 $k$ 的掌握度，对于每种算法，应优先选择提升值 $b_i$ 最大的题目以最小化数量。关键约束是：不能连续做两道相同知识点的题，这要求最终选择的多重集能排列成相邻不同色的序列。

### 核心思路

**Step 1：求每种算法的最小题目数**
对每种算法 $t$，收集其所有 $b_i$，降序排序，计算前缀和。找到最小的 $c_t$ 使得前缀和 $\ge k$。若某算法的所有题之和仍 $<k$，则无解，输出 $-1$。

**Step 2：检查能否排成合法序列**
设 $total = \sum c_t$，$mx = \max c_t$。定理：一个多重集能排列成相邻不同的序列当且仅当 $mx \le total - mx + 1$（即数量最多的类型不超过总数的一半向上取整）。

若条件满足，答案即为 $total$。

**Step 3：补充题目**
若 $mx > total - mx + 1$，则需从非 $mx$ 的类型中额外选题目（即使已满足掌握需求），以增加 $total$ 使条件成立。需要额外数量：
\[
extra = mx - (total - mx + 1) = 2\cdot mx - total - 1
\]
遍历所有非最大类型，将其剩余的题目加入候选池（即该类型总题数 $- c_t$），统计可补充的总数。若够，答案 $= total + extra$；否则 $-1$。

### 算法流程

1. 按 $a_i$ 分组，每组内 $b_i$ 降序排序，求前缀和。
2. 对每组计算最小 $c_t$，同时检测不可行情况。
3. 统计 $total$、$mx$ 以及各非最大类型的剩余题数。
4. 若 $mx \le total - mx + 1$，输出 $total$。
5. 否则计算 $extra$，检查剩余是否足够，输出相应结果。

### 复杂度分析

- **时间复杂度**：$O(n\log n)$，排序各组的 $b_i$。
- **空间复杂度**：$O(n)$。

## 参考代码

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int m, n, k;
    cin >> m >> n >> k;
    
    vector<int> a(n), b(n);
    for (int i = 0; i < n; i++) cin >> a[i];
    for (int i = 0; i < n; i++) cin >> b[i];
    
    vector<vector<int>> groups(m + 1);
    for (int i = 0; i < n; i++) {
        groups[a[i]].push_back(b[i]);
    }
    
    for (int t = 1; t <= m; t++) {
        sort(groups[t].rbegin(), groups[t].rend());
    }
    
    vector<int> c(m + 1, 0);
    vector<ll> rem(m + 1, 0);
    
    for (int t = 1; t <= m; t++) {
        ll sum = 0;
        for (int val : groups[t]) {
            sum += val;
            c[t]++;
            if (sum >= k) break;
        }
        if (sum < k) {
            cout << -1 << '\n';
            return 0;
        }
        rem[t] = (ll)groups[t].size() - c[t];
    }
    
    ll total = 0;
    int mx = 0;
    for (int t = 1; t <= m; t++) {
        total += c[t];
        mx = max(mx, c[t]);
    }
    
    if (mx <= total - mx + 1) {
        cout << total << '\n';
        return 0;
    }
    
    ll need = (ll)mx - (total - mx + 1);  // how many extra needed
    ll extra = 0;
    for (int t = 1; t <= m; t++) {
        if (c[t] < mx) extra += rem[t];
    }
    
    if (extra >= need) {
        cout << total + need << '\n';
    } else {
        cout << -1 << '\n';
    }
    
    return 0;
}
```
