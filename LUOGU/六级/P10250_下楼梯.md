# [GESP样题 六级] 下楼梯

## 题目描述

顽皮的小明发现，下楼梯时每步可以走 $1$ 个台阶、$2$ 个台阶或 $3$ 个台阶。现在一共有 $N$ 个台阶，你能帮小明算算有多少种方案吗？

## 输入格式

一行一个整数 $N$。

## 输出格式

一行一个整数表示答案。

## 样例

### 样例 1

```
输入：
4

输出：
7
```

### 样例 2

```
输入：
10

输出：
274
```

## 提示

对全部的测试点，保证 $1\le N\le 60$。

## 解题思路

### 问题分析

这是一个经典的"上楼梯"问题的变种——下楼梯。从第 $N$ 阶下到第 $0$ 阶，每一步可以走 $1$、$2$ 或 $3$ 个台阶。方案数与从第 $0$ 阶上到第 $N$ 阶的上楼方案数完全相同，只是方向相反。

### 核心思路

定义 $f(i)$ 为从第 $i$ 阶下到第 $0$ 阶的方案数。考虑第一步可以走 $1$、$2$ 或 $3$ 阶，得到递推关系：

$$
f(i) = f(i-1) + f(i-2) + f(i-3)
$$

边界条件：
- $f(0)=1$（已在平地，什么都不做就是一种方案）。
- $f(1)=1$（只能走 $1$ 阶）。
- $f(2)=2$（$1+1$ 或 $2$）。

$N\le 60$，使用 `unsigned long long` 即可避免溢出。最大答案约为 $1.2\times 1.839^{60}\approx 2.5\times 10^{14}$，在 $2^{64}$ 范围内。

### 算法流程

1. 读入 $N$。
2. 初始化 $f[0]=1$，$f[1]=1$，$f[2]=2$。
3. 从 $3$ 到 $N$ 循环计算 $f[i]=f[i-1]+f[i-2]+f[i-3]$。
4. 输出 $f[N]$。

### 复杂度分析

- **时间复杂度**：$O(N)$。
- **空间复杂度**：$O(1)$（只需保留最后三个值）。

## 参考代码

```cpp
#include <bits/stdc++.h>
using namespace std;

using ull = unsigned long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    cin >> n;
    
    if (n == 0) { cout << 1 << '\n'; return 0; }
    if (n == 1) { cout << 1 << '\n'; return 0; }
    if (n == 2) { cout << 2 << '\n'; return 0; }
    
    ull a = 1, b = 1, c = 2;  // f(0), f(1), f(2)
    for (int i = 3; i <= n; i++) {
        ull d = a + b + c;
        a = b;
        b = c;
        c = d;
    }
    cout << c << '\n';
    
    return 0;
}
```
