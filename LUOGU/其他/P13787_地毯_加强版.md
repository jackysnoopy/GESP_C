# 地毯 加强版

## 题目描述

在 $n\times n$ 的格子上有 $m$ 个地毯。

给出这些地毯的信息，问每个点被多少个地毯覆盖。

## 输入格式

第一行，两个正整数 $n,m$。意义如题所述。

接下来 $m$ 行，每行两个坐标 $(x_1,y_1)$ 和 $(x_2,y_2)$，代表一块地毯，左上角是 $(x_1,y_1)$，右下角是 $(x_2,y_2)$。

## 输出格式

为了减少输出量，设 $F_{i,j}$ 表示 $(i,j)$ 这个格子被多少个地毯覆盖，你只需要输出 $\sum_{i=1}^n\sum_{j=1}^n (i+j)\oplus F_{i,j}$ 的值。注意这个值可能会超过 $2^{31}$。

## 样例 #1

**输入**
```
5 3
2 2 3 3
3 3 5 5
1 2 1 4
```

**输出**
```
146
```

## 提示

对于 $50\%$ 的数据，有 $n,m\le 5000$。

对于 $100\%$ 的数据，有 $n\le 5000$，$m\le 2\times 10^5$。

## 解题思路

### 问题分析

$n \times n$ 网格，$m$ 个地毯。与 P3397 类似，但可能数据范围更大。输出每个格子被覆盖的次数。

### 核心思路

**二维差分**直接应用。与 P3397（地毯）完全相同，使用二维差分 $O(1)$ 标记每个地毯，然后二维前缀和恢复。

### 算法流程

1. 初始化二维差分数组 $d$。
2. 对于每个地毯，做二维差分的四个标记。
3. 二维前缀和恢复，输出每个格子的值。

### 复杂度分析

- **时间复杂度：** $O(n^2 + m)$。
- **空间复杂度：** $O(n^2)$。

### 要点

- 与 P3397 的区别在于数据范围可能更大，需要更注意数组大小和内存使用。
- 如果 $n$ 较大但 $m$ 较小，可以用稀疏方式存储。

## 参考代码

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, m;
    cin >> n >> m;
    vector<vector<int>> d(n + 2, vector<int>(n + 2, 0));
    while (m--) {
        int x1, y1, x2, y2;
        cin >> x1 >> y1 >> x2 >> y2;
        d[x1][y1]++; d[x1][y2+1]--; d[x2+1][y1]--; d[x2+1][y2+1]++;
    }
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= n; j++) {
            d[i][j] += d[i-1][j] + d[i][j-1] - d[i-1][j-1];
            cout << d[i][j] << " \n"[j == n];
        }
    }
    return 0;
}
```
