# Full Tank?

## 题目描述
带油量状态的最短路。

## 输入格式
多组城市/道路/油站。

## 输出格式
每次查询最小花费。

## 样例
### 样例 1
```
4 4 1
0 1 1 1
1 2 1 2
2 3 1 2
3 4 1 2
3
0 1 1
0 2 2
0 3 3
```
```
4
4
5
```

## 解题思路
### 问题分析
本题要求从起点到终点的最小加油费用。汽车油箱容量为 $P$，到达每个城市可以加 $1 \sim P$ 单位的油，每单位油的价格为该城市的油价。道路有距离（即耗油量）。

### 核心思路
1. **分层最短路**：将状态定义为 `(城市, 当前油量)`，构建分层图。
2. 两种转移：
   - **加油**：在当前城市加 1 单位油，花费该城市油价，状态变为 `(城市, 油量+1)`
   - **行驶**：沿道路行驶到相邻城市，消耗等量油，油量减少
3. 使用 Dijkstra 算法在分层图上求最短路，起点状态为 `(起点, 0)`，每次查询目标为 `(终点, 任意油量)` 的最小值。

### 算法流程
1. 读入城市、道路信息和油价
2. 构建分层图：节点为 `(城市, 油量)`，边包括加油边和行驶边
3. 对每个查询，运行 Dijkstra 从 `(s, 0)` 出发
4. 取 `(t, 0) ~ (t, P)` 中的最小距离作为答案

### 复杂度分析
- 时间复杂度：$O(Q \times P \times N \log(PN))$，其中 $Q$ 为查询数，$P$ 为油箱容量，$N$ 为城市数
- 空间复杂度：$O(PN)$
