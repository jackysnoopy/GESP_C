# Pushing Boxes

## 题目描述
推箱子求最短步数。

## 输入格式
多组地图。

## 输出格式
最少步数或 impossible。

## 样例
### 样例 1
```
2 1
#.
@$
0 0
```
```
1
```

## 解题思路
### 问题分析
经典推箱子（Sokoban）问题的变体，要求求出将箱子推到目标位置的最少步数。地图中 `@` 为推箱子的人，`$` 为箱子，`.` 为目标位置，`#` 为墙壁。

### 核心思路
1. **双层 BFS**：状态为 `(人位置, 箱子位置)`，对状态空间做 BFS。
2. 人可以在不推动箱子的情况下自由移动（只走空地和目标点），这部分也用 BFS 求最短路径。
3. 当人到达箱子相邻位置且另一侧为空地时，可以推箱子一格。
4. 使用 `vis[people][box]` 去重，避免重复访问同一状态。

### 算法流程
1. 读入地图，找到人、箱子、目标的位置
2. BFS 搜索 `(人, 箱子)` 状态对
3. 对每个状态，先 BFS 求人能到达的所有空地
4. 检查人是否能到达箱子相邻位置，若能则尝试推箱子
5. 若箱子到达目标位置，输出步数
6. 若搜索完无法到达目标，输出 "impossible"

### 复杂度分析
- 时间复杂度：$O(R^2 \times C^2 \times RC)$，其中 $R, C$ 为行列数
- 空间复杂度：$O(R^2 \times C^2)$
