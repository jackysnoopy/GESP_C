# 八数码难题

## 题目描述
3x3 滑块拼图最小步数。

## 输入格式
9 位状态字符串。

## 输出格式
最少步数。

## 样例
### 样例 1
```
283104765
```
```
4
```

## 解题思路
### 问题分析
经典八数码问题。3×3 棋盘上有 8 个编号方块和一个空格（用 0 表示），每次可以将空格与相邻方块交换。目标是将初始状态变换为目标状态 `123456780`。

### 核心思路
1. **双向 BFS**：从初始状态和目标状态同时搜索，相遇时即为最短路径。
2. **A* 搜索**：使用 Manhattan 距离作为启发函数，优先扩展 $f(n) = g(n) + h(n)$ 最小的状态。
3. **状态编码**：将 9 位排列编码为康托展开（Cantor Expansion）对应的整数，用于哈希判重。
4. 预计算目标状态中每个数字的位置，快速计算 Manhattan 距离之和。

### 算法流程
1. 读入初始状态字符串
2. 计算初始状态的康托展开值，标记为目标状态的康托展开值
3. 双向 BFS 或 A* 搜索
4. 每次扩展时找到空格位置，尝试上下左右四个方向交换
5. 到达目标状态时输出步数

### 复杂度分析
- 时间复杂度：双向 BFS 最坏 $O(9!/2)$；A* 通常更优
- 空间复杂度：$O(9!)$，使用康托展开数组判重
