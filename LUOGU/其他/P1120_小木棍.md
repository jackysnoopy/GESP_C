# 小木棍

## 题目描述
将砍断的等长小木棍拼回原长，求原长最小可能。

## 输入格式
第一行 n，随后 n 行每行一个 <=50 的整数。

## 输出格式
原长最小值。

## 样例
### 样例 1
```
9
5
2
1
5
2
1
5
2
1
```
```
6
```

## 解题思路
### 问题分析
将若干根小木棍重新拼接成若干根等长的原木棍，求原木棍的最小长度。每根小木棍必须恰好使用一次，每根原木棍由若干小木棍拼接而成，总长度等于原木棍长度。

### 核心思路
1. **搜索 + 剪枝**：枚举原木棍长度 $L$（$L$ 为所有木棍总和的因数，且 $L \ge$ 最长木棍长度）。
2. 对每个 $L$，DFS 尝试将木棍分配到原木棍中，每根原木棍凑满 $L$ 后开始下一根。
3. **剪枝策略**：
   - 按长度降序排列，先放长木棍
   - 若当前原木棍剩余空间等于某根木棍长度，优先放入
   - 跳过相同长度的木棍（避免重复搜索）
   - 剩余最长木棍 > 当前剩余空间时剪枝

### 算法流程
1. 读入所有小木棍长度，计算总和 $S$
2. 将长度降序排序
3. 从小到大枚举 $L$（$L$ 为 $S$ 的因数且 $\ge$ 最大木棍长度）
4. 对每个 $L$，DFS 尝试拼接，检查是否能恰好拼成 $S/L$ 根原木棍
5. 第一个成功的 $L$ 即为答案

### 复杂度分析
- 时间复杂度：因数枚举 $O(\sqrt{S})$ × 搜索 $O(K^N)$，$K$ 为当前原木棍数，实际因剪枝远小于此
- 空间复杂度：$O(N)$
