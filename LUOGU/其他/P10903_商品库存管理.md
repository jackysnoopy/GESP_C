# [蓝桥杯 2024 省 C] 商品库存管理

## 题目描述

在库存管理系统中，跟踪和调节商品库存量是关键任务之一。小蓝经营的仓库中存有多种商品，这些商品根据类别和规格被有序地分类并编号，编号范围从 $1$ 至 $n$。初始时，每种商品的库存量均为 $0$。

为了高效地监控和调整库存量，小蓝的管理团队设计了 $m$ 个操作，每个操作涉及到一个特定的商品区间，即一段连续的商品编号范围（例如区间 $[L, R]$）。执行这些操作时，区间内每种商品的库存量都将增加 $1$。然而，在某些情况下，管理团队可能会决定不执行某些操作，使得这些操作涉及的商品区间内的库存量不会发生改变，维持原有的状态。

现在，管理团队需要一个评估机制，来确定如果某个操作未被执行，那么最终会有多少种商品的库存量为 $0$。对此，请你为管理团队计算出，对于每个操作，如果不执行该操作而执行其它操作，库存量为 $0$ 的商品的种类数。

## 输入格式

输入的第一行包含两个整数 $n$ 和 $m$，分别表示商品的种类数和操作的个数。

接下来的 $m$ 行，每行包含两个整数 $L$ 和 $R$，表示一个操作涉及的商品区间。

## 输出格式

输出 $m$ 行，每行一个整数，第 $i$ 行的整数表示如果不执行第 $i$ 个操作，则最终库存量为 $0$ 的商品种类数。

## 样例 #1

**输入**
```
5 3
1 2
2 4
3 5
```

**输出**
```
1
0
1
```

## 提示

**【样例说明】**

## 解题思路

### 问题分析

$n$ 个商品，$m$ 个库存管理操作。每个操作在区间 $[l,r]$ 上增加 $1$ 库存。对于每个操作 $i$，问如果去掉该操作，最终库存为 $0$ 的位置有多少个。

### 核心思路

先用差分数组处理所有操作，得到每个位置的最终库存 $cnt[i]$。对于每个询问 $i$，去掉操作 $i$ 后：
- 原本为 $0$ 的位置仍然为 $0$（只要该位置不是仅被操作 $i$ 覆盖）。
- 原本为 $1$ 的位置，如果仅由操作 $i$ 覆盖，就会变为 $0$。

所以对于操作 $i$，它影响的区间 $[l_i, r_i]$ 内，那些被恰好 $1$ 个操作覆盖（且就是 $i$）的位置会从 $1$ 变为 $0$。总答案 = 全局 $0$ 的个数 + 区间内被恰好 $1$ 个操作覆盖的位置数。

### 算法流程

1. 用差分数组处理所有 $m$ 个操作，得到 $cnt[i]$。
2. 统计全局 $0$ 的个数 $zero$。
3. 构建前缀和 $one[i]$ 表示前 $i$ 个位置中 $cnt = 1$ 的个数。
4. 对于每个操作 $i$，答案为 $zero + (one[r_i] - one[l_i-1])$。

### 复杂度分析

- **时间复杂度：** $O(n + m)$。
- **空间复杂度：** $O(n)$。

### 要点

- 关键观察：去掉操作 $i$ 后，只有那些原来 $cnt=1$ 且在操作 $i$ 区间内的位置会变为 $0$。
- 如果某个位置 $cnt > 1$，去掉一个操作不会让它变 $0$。

## 参考代码

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, m;
    cin >> n >> m;
    vector<int> l(m+1), r(m+1), d(n+2, 0);
    for (int i = 1; i <= m; i++) {
        cin >> l[i] >> r[i];
        d[l[i]]++; d[r[i]+1]--;
    }
    vector<int> cnt(n+1, 0);
    int zero = 0;
    for (int i = 1; i <= n; i++) {
        cnt[i] = cnt[i-1] + d[i];
        if (cnt[i] == 0) zero++;
    }
    vector<int> one(n+1, 0);
    for (int i = 1; i <= n; i++) one[i] = one[i-1] + (cnt[i] == 1);
    for (int i = 1; i <= m; i++) {
        int c = one[r[i]] - one[l[i] - 1];
        cout << zero + c << "\n";
    }
    return 0;
}
```
