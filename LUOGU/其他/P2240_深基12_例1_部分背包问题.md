# 【深基12.例1】部分背包问题

## 题目描述

阿里巴巴走进了装满宝藏的藏宝洞。藏宝洞里面有 $N(N \le 100)$ 堆金币，第 $i$ 堆金币的总重量和总价值分别是 $m_i,v_i(1\le m_i,v_i \le 100)$。阿里巴巴有一个承重量为 $T(T \le 1000)$ 的背包，但并不一定有办法将全部的金币都装进去。他想装走尽可能多价值的金币。所有金币都可以随意分割，分割完的金币重量价值比（也就是单位价格）不变。请问阿里巴巴最多可以拿走多少价值的金币？

## 输入格式

第一行两个整数 $N,T$。

接下来 $N$ 行，每行两个整数 $m_i,v_i$。

## 输出格式

一个实数表示答案，输出两位小数。

## 样例

### 样例输入1
```
4 50
10 60
20 100
25 100
15 45
```

### 样例输出1
```
240.00
```

## 提示

## 解题思路

### 问题分析

经典**部分背包问题**（Fractional Knapsack）：$N$ 个物品各有重量 $m_i$ 和价值 $v_i$，背包容量为 $T$，物品可以分割（取部分），求最大总价值。

### 核心思路

**贪心算法**：按**单位价值**（价值/重量 = $v_i / m_i$）从大到小排序，依次贪心取：

1. 若当前物品能完全装入（$m_i \le$ 剩余容量），则全部取走。
2. 若不能完全装入，则取能装下的部分：$\frac{v_i}{m_i} \times$ 剩余容量。

**正确性证明**：部分背包可以取任意比例，按单位价值降序取一定是最优的。若存在更优解，则必有一处取了单位价值较低的物品而未取单位价值较高的物品，交换后总价值不会减少。

### 算法流程

1. 读入 $N, T$ 和每堆金币的重量 $m_i$、价值 $v_i$。
2. 按 $v_i / m_i$ 降序排序。
3. 遍历排序后的物品：
   - 若 $m_i \le$ 剩余容量，全部取走，剩余容量 $-= m_i$，总价值 $+= v_i$。
   - 否则取部分，总价值 $+= \frac{v_i}{m_i} \times$ 剩余容量，剩余容量 $= 0$。
4. 输出总价值，保留两位小数。

### 复杂度分析

- **时间复杂度**：$O(N \log N)$，排序主导。
- **空间复杂度**：$O(N)$，存储物品信息。

## 参考代码

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <cstdio>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, t;
    cin >> n >> t;
    vector<pair<int, int>> items(n);
    for (int i = 0; i < n; i++) {
        cin >> items[i].first >> items[i].second;
    }

    sort(items.begin(), items.end(), [](const pair<int, int>& a, const pair<int, int>& b) {
        return (double)a.second / a.first > (double)b.second / b.first;
    });

    double ans = 0;
    int rem = t;
    for (int i = 0; i < n && rem > 0; i++) {
        int w = items[i].first, v = items[i].second;
        if (w <= rem) {
            ans += v;
            rem -= w;
        } else {
            ans += (double)v / w * rem;
            rem = 0;
        }
    }

    printf("%.2f\n", ans);
    return 0;
}
```
