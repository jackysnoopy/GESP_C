# [NOIP 2018 提高组] 铺设道路

## 题目描述

春春是一名道路工程师，负责铺设一条长度为 $n$ 的道路。 

铺设道路的主要工作是填平下陷的地表。整段道路可以看作是 $n$ 块连续的区域，一开始，第 $i$ 块区域下陷的深度为 $d_i$ 。 

春春每天可以选择一段连续区间 $[L,R]$ ，填充这段区间中的每块区域，让其下陷深度减少 $1$。在选择区间时，需要保证，区间内的每块区域在填充前下陷深度均不为 $0$ 。 

春春希望你能帮他设计一种方案，可以在最短的时间内将整段道路的下陷深度都变为 $0$ 。

## 输入格式

输入文件包含两行，第一行包含一个整数 $n$，表示道路的长度。 第二行包含 $n$ 个整数，相邻两数间用一个空格隔开，第 $i$ 个整数为 $d_i$ 。

## 输出格式

输出文件仅包含一个整数，即最少需要多少天才能完成任务。

## 样例 #1

**输入**
```
6   
4 3 2 5 3 5
```

**输出**
```
9
```

## 提示

【样例解释】

一种可行的最佳方案是，依次选择： 
$[1,6]$、$[1,6]$、$[1,2]$、$[1,1]$、$[4,6]$、$[4,4]$、$[4,4]$、$[6,6]$、$[6,6]$。   

【数据规模与约定】

对于 $30\%$ 的数据，$1 ≤ n ≤ 10$ ；    
对于 $70\%$ 的数据，$1 ≤ n ≤ 1000$ ；     
对于 $100\%$ 的数据，$1 ≤ n ≤ 100000 , 0 ≤ d_i ≤ 10000$ 。

## 解题思路

### 问题分析

道路长度为 $n$，第 $i$ 块区域下陷深度为 $d_i$。每天可以选择一段连续区间填充 $1$ 单位深度。求最少需要多少天将所有区域填平。

### 核心思路

**差分贪心。** 设 $d_0 = 0$，则答案为 $\sum_{i=1}^{n} \max(0, d_i - d_{i-1})$。

**直观理解：** 从左到右扫描，如果当前深度比前一区域深，则必须增加额外的填充操作来覆盖这段增量。这与括号匹配的思想类似。

### 算法流程

1. 读入 $n$ 和数组 $d$。
2. $ans = d[0]$（第一个区域需要的操作数）。
3. 遍历 $i=1..n-1$，如果 $d[i] > d[i-1]$，$ans += d[i] - d[i-1]$。
4. 输出 $ans$。

### 复杂度分析

- **时间复杂度：** $O(n)$。
- **空间复杂度：** $O(1)$ 或 $O(n)$。

### 正确性证明

每次操作选择一个区间减 $1$。在差分视角下，原数组的差分数组 $diff[i] = d[i] - d[i-1]$（$d_0=0$）。一次区间操作相当于 $diff[l] -= 1$，$diff[r+1] += 1$。我们的目标是将所有 $diff$ 变为 $0$。正差分需要被负差分抵消，每个正差分 $diff[i] > 0$ 都需要一次区间开始操作，所以总操作数 = 正差分之和。

### 扩展

本题与 P3078（Poker Hands）完全一致，也与 AcWing 的"最高的牛"有相似思路。

## 参考代码

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n;
    cin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];
    long long ans = a[0];
    for (int i = 1; i < n; i++) {
        if (a[i] > a[i - 1]) ans += a[i] - a[i - 1];
    }
    cout << ans << "\n";
    return 0;
}
```
