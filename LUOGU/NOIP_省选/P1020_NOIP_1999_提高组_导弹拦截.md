# [NOIP 1999 提高组] 导弹拦截

## 题目描述

某国为了防御敌国的导弹袭击，发展出一种导弹拦截系统。但是这种导弹拦截系统有一个缺陷：虽然它的第一发炮弹能够到达任意的高度，但是以后每一发炮弹都不能高于前一发的高度。某天，雷达捕捉到敌国的导弹来袭。由于该系统还在试用阶段，所以只有一套系统，因此有可能不能拦截所有的导弹。

   
输入导弹依次飞来的高度，计算这套系统最多能拦截多少导弹，如果要拦截所有导弹最少要配备多少套这种导弹拦截系统。

## 输入格式

一行，若干个整数，中间由空格隔开。

## 输出格式

两行，每行一个整数，第一个数字表示这套系统最多能拦截多少导弹，第二个数字表示如果要拦截所有导弹最少要配备多少套这种导弹拦截系统。

## 样例

### 样例输入
```
389 207 155 300 299 170 158 65
```

### 样例输出
```
6
2
```

## 提示

对于前 $50\%$ 数据，满足导弹的个数不超过 $10^4$ 个。该部分数据总分共 $100$ 分。可使用 $\mathcal O(n^2)$ 做法通过。  
对于后 $50\%$ 的数据，满足导弹的个数不超过 $10^5$ 个。该部分数据总分也为 $100$ 分。请使用 $\mathcal O(n\log n)$ 做法通过。

对于全部数据，满足导弹的高度为正整数，且不超过 $5	imes 10^4$。

## 解题思路

### 问题分析

经典 LIS 问题：第一问求最长不上升子序列（LNIS），第二问求最少拦截系统数（= 最长严格递增子序列 LIS 长度）。

### 核心思路

第一问：贪心 + patience sorting，维护 piles 数组。第二问：Dilworth 定理——最少链覆盖 = 最长反链长度。

### 算法流程

1. 第一问：对每个高度，用 upper_bound 在 piles 中找第一个 ≤ 它的位置（从大到小），更新或追加
2. 第二问：对每个高度，用 lower_bound 在 piles 中找第一个 ≥ 它的位置，更新或追加
3. 分别输出两个 piles 的大小

### 复杂度分析

- 时间复杂度：O(n·log(n))
- 空间复杂度：O(n)

### 关键技巧

- 第一问用 upper_bound（非严格递减），第二问用 lower_bound（严格递增）
- Dilworth 定理将问题转化为 LIS
