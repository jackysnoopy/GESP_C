# [GESP202403 四级] 做题

## 题目描述

小杨同学为了提高自己的实力制定了做题计划，在第 $i$ 天时，他必须要完成 $i$ 道题，否则他就会偷懒。

小杨同学现在找到了一个题库，一共有 $n$ 套题单，每一套题单中有一定数量的题目。但是他十分挑剔，每套题单他只会使用一次，每一天也只能使用一套题单里的题目，之后那套题单就会弃之不用。对于每套题单，他不必完成题单内所有的题。

那么问题来了，小杨同学最多做题几天才偷懒呢？

## 输入格式

第一行，一个整数为 $n$，表示有多少套题单。

第二行 $n$ 个整数 $a_i$，分别表示每套题单有多少道题。

## 输出格式

输出一行一个整数表示答案。

## 样例

### 样例 #1

**输入**
```
4
3 1 4 1
```

**输出**
```
2
```

**解释**

小杨同学有以下做题方案：
- 第 $1$ 天，从题单 $4$ 道题的题单中做 $1$ 道题。
- 第 $2$ 天，从题单 $3$ 道题的题单中做 $2$ 道题。
- 到了第 $3$ 天，还想做题，只剩 $1$ 道题的题单，需要做 $3$ 道题，不够，于是偷懒。
所以最多做 $2$ 天。

## 提示

**数据范围**

$1 \le n \le 10^5$，$1 \le a_i \le 10^9$。

## 解题思路

### 问题分析

我们每天需要完成 $i$ 道题，只能从一套题单中取题，且每套题单只能使用一次。目标：最大化的天数。

### 核心思路

贪心。将题单按题目数量从小到大排序，尽量先用题目数量少的题单来完成当前天数的要求。

### 算法流程

1. 对 $a_i$ 从小到大排序。
2. 设置当前天数 $day = 1$。
3. 遍历排序后的题单：
   - 如果 $a_i \ge day$，说明这套题单可以满足第 $day$ 天的要求，$day$ 加 $1$。
   - 否则，跳过这套题单。
4. 最终答案为 $day - 1$。

### 复杂度分析

- **时间复杂度：** $O(n \log n)$，主要来自排序。
- **空间复杂度：** $O(n)$。

## 参考代码

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n;
    cin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];
    sort(a.begin(), a.end());
    int day = 1;
    for (int x : a) {
        if (x >= day) day++;
    }
    cout << day - 1 << "\n";
    return 0;
}
```
