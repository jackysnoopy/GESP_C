# 【MX-X18-T4】「FAOI-R6」绿茶

## 题目描述

> 天空的颜色刚刚好 / 绿茶的香味随着风在飘  
> 我说喜欢有这样的美好 / 在围绕 / 不去管花落了多少

你有两个 $n$ 位二进制数 $A,B$（可能有前导 0）和一个非负整数序列 $c_0, c_1, \ldots, c_{n - 1}$。

你可以进行若干次如下操作：
- 选择满足 $\lvert A-C\rvert = 2^k$ 且 $k < n$ 的非负整数 $C, k$，然后将 $A$ 变为 $A \mathbin{\mathrm{or}} C$，代价为 $c_k$。

其中 $\mathbin{\mathrm{or}}$ 表示[二进制按位或](https://oi-wiki.org/math/bit/#%E4%B8%8E%E6%88%96%E5%BC%82%E6%88%96)。

你需要通过若干次操作使得 $A$ 变为 $B$，求出最小的代价和。若无解则输出 $-1$。

## 输入格式

**本题输入包含多组数据。**

第一行，一个整数 $T$，表示数据组数。对于每组数据：

- 第一行，一个正整数 $n$。
- 第二行，一个长度为 $n$ 的 01 串，表示 $A$ 二进制下从高到低的数位，可能有前导 0。
- 第三行，一个长度为 $n$ 的 01 串，表示 $B$ 二进制下从高到低的数位，可能有前导 0。
- 第四行，$n$ 个非负整数 $c_{n-1},c_{n-2},\ldots,c_0$，**注意此处是倒序的。**

## 输出格式

对于每组测试数据，输出一行，一个整数，表示答案。

**【样例解释】**

对于第二组数据，两次操作分别为：
- 选择 $C=1$，$\lvert A-C\rvert=2^0$，代价为 $c_0=3$，操作后 $A=1$；
- 选择 $C=5$，$\lvert A-C\rvert=2^2$，代价为 $c_2=1$，操作后 $A=5$；

可以证明不存在代价更小的操作方法，故答案为 $4$。

## 参考代码

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;

    while (T--) {
        int n;
        cin >> n;

        string A, B;
        cin >> A >> B;

        vector<int> c(n);
        for (int i = 0; i < n; i++) cin >> c[i];

        long long ans = 0;
        bool possible = true;

        for (int i = 0; i < n; i++) {
            if (A[i] == '0' && B[i] == '1') {
                ans += c[n - 1 - i];
            } else if (A[i] == '1' && B[i] == '0') {
                possible = false;
                break;
            }
        }

        cout << (possible ? to_string(ans) : "-1") << "\n";
    }

    return 0;
}
```

## 解题思路

### 问题分析
给定两个 $n$ 位二进制数 $A$ 和 $B$，以及代价数组 $c_0, c_1, \ldots, c_{n-1}$。每次操作选择满足 $|A-C|=2^k$ 的非负整数 $C$ 和 $k$，将 $A$ 变为 $A \mathbin{\mathrm{or}} C$，代价为 $c_k$。求将 $A$ 变为 $B$ 的最小代价，若不可能则输出 $-1$。

### 核心思路
由于操作是按位或（OR），只能将 $A$ 的某一位从 $0$ 置为 $1$，不能将 $1$ 置为 $0$。因此若 $A$ 的某位为 $1$ 而 $B$ 的对应位为 $0$，则无解。对于 $A$ 为 $0$ 且 $B$ 为 $1$ 的位，每次操作可独立地将该位置 $1$，代价为 $c_k$（$k$ 为对应位权）。总代价为所有需要置 $1$ 的位的代价之和。

### 算法流程
1. 读入 $n$、字符串 $A$、字符串 $B$ 和代价数组 $c$
2. 从高位到低位遍历每一位 $i$：
   - 若 $A[i]=0$ 且 $B[i]=1$，累加代价 $c[n-1-i]$
   - 若 $A[i]=1$ 且 $B[i]=0$，标记无解并跳出
3. 若有解，输出累加的总代价；否则输出 $-1$

### 复杂度分析
- **时间复杂度**：$O(n)$。对每组数据只需遍历 $n$ 位一次
- **空间复杂度**：$O(n)$。存储代价数组 $c$，其余为 $O(1)$
