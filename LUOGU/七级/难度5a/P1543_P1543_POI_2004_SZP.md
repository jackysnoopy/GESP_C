# P1543 [POI 2004] SZP

## 题目分析
- `n` 个同学，每人监视另一人 `a_k`，构成有向图 `k → a_k`。
- 选最多人，使得被选中的每人都"至少有一名监视她的（前驱）没被选中"。
- 没人被监视的人不能被选。
- 数据范围 `n ≤ 10^6`，注意空间限制 32MB。

## 模型转换
> 对每个被选中的 `v`，它的入边集合中至少存在一个 `u` 没被选。

设每个 `v` 的入度（即监视它的人数）为 `in[v]`。

### 拓扑剥叶子
- 若 `in[v] = 0`：没人监视 `v` → `v` 不能选，且 `v` 必然不选。
- `v` 不被选 → `v` 监视的 `a[v]` 至少有一个未被选的前驱（即 `v` 自己）→ `a[v]` 可以选！
- 选 `a[v]` 后，`a[v]` 监视 `a[a[v]]`，对 `a[a[v]]` 的入边来说，`a[v]` 已被占用，`in[a[a[v]]] -= 1`，若变 0 又递归处理。

实现：
1. 把 `in[i] == 0` 的点入队 → 它们都不选。
2. 出队 `u` → 标记它监视的 `v = a[u]` 为"被选"，答案 +1，标记 `v` 已访问。
3. 对 `w = a[v]`，若未被访问，`in[w]--`；若变 0 入队（表示 `w` 也不能选）。
4. 拓扑结束后，剩下的就是若干**纯环**。

### 处理纯环
环长 `L`，环上每个点都被恰好一人监视（即环上前驱）。环上要选最多的点，且每个被选点都需要前驱不被选 → 环上不能两个相邻都被选 → 最多 `L-1` 个（不选其中任意 1 个，其它全选）。

合计答案 = 拓扑阶段 + Σ(每个环 `L-1`)。

## 实现细节
- 静态数组 `a[]`、`in_deg[]`、`vis[]` 大小 1e6+5。
- `vis` 标记任何"已确定不选/已选/已遍历"的点，避免重复。
- 注意：当 `vis[v]` 已被前一拓扑路径选中时不重复增加 `ans`。

## 复杂度
- 时间 `O(n)`。
- 空间 `O(n)`（int + int + bool ≈ 9MB << 32MB）。

## 样例验证
输入：1→2, 2→3, 3→1, 4→3, 5→6, 6→5。
- `in[]`: 1:1, 2:1, 3:2, 4:0, 5:1, 6:1。
- 4 入度 0，4 不选 → a[4]=3 被选，ans=1。处理 a[3]=1, in[1]-- = 0 入队。
- 处理 1：1 不选 → a[1]=2 被选（ans=2），但 2 已被前面影响……重新走一遍：1 出队，a[1]=2 vis 否 → 选 2，ans=2，a[2]=3 已 vis 跳过。
- 队列空。剩 5,6 是一个 2 环，长度 2，加 1，总 ans=3。
输出 `3`，正确。

## 解题思路

### 问题分析
给定 n 个同学的监视关系（每人监视另一人），构成有向图。需要选择最多的同学，使得每个被选中的同学至少有一个监视她的人没被选中。

### 核心思路
将问题分解为两个阶段处理：
1. **拓扑剥叶子阶段**：从入度为 0 的节点开始，这些节点必然不能选，但可以使其监视的节点被选
2. **纯环处理阶段**：拓扑结束后剩余的都是环，每个长度为 L 的环最多选 L-1 个节点

### 算法流程
1. 统计每个节点的入度
2. 将入度为 0 的节点入队（这些节点不能选）
3. BFS 处理：出队节点 u，标记其监视的节点 v 为被选，ans++
4. 对 v 监视的节点 w，入度减 1，若变为 0 则入队
5. 拓扑结束后，遍历未访问节点，计算每个环的长度，累加 len-1

### 复杂度分析
- 时间复杂度：O(n)，每个节点和边只处理一次
- 空间复杂度：O(n)，用于存储数组和队列
