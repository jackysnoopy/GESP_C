# [BalticOI 2003] Register Allocation

## 题目描述

给定表达式树和 N 个寄存器，求最小求值代价。叶节点从内存加载（代价 Cl），中间节点执行运算（给定代价），存结果到内存代价 Cs。

## 输入格式

第一行 N（寄存器数）。
第二行 Cl Cs（加载/存储代价）。
接下来描述表达式树：每个节点先给子节点数 K，若 K>0 再给运算代价，然后递归描述 K 棵子树。

## 输出格式

第一行最小代价。
接下来每行：节点编号和 0/1（0=存内存，1=保留寄存器），按执行顺序。

## 样例

输入：
```
2
3 2
2
10
2
15
0
0
2
5
0
0
```
输出：
```
47
2 0
5 1
1 1
```

## 提示

- 1 <= N <= 100
- 节点数 M <= 10000

## 解题思路

### 问题分析

表达式树求值，寄存器有限。需要决定每个中间结果是保留还是存内存。

### 核心思路

树形 DP：对每个节点，计算在给定可用寄存器数下的最小代价。

### 算法流程

1. 后序遍历表达式树
2. 对每个节点，枚举保留多少个子结果在寄存器中
3. 贪心选择代价最小的方案

### 复杂度分析

- 时间复杂度：O(M * N)
- 空间复杂度：O(M)
