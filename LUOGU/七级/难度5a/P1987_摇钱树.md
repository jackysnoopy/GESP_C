# 摇钱树

## 题目描述

Cpg 正在游览一个梦中之城，在这个城市中有 $n$ 棵摇钱树。这下，可让 Cpg 看傻了。可是 Cpg 只能在这个城市中呆 $k$ 天，但是现在摇钱树已经成熟了，每天每棵都会掉下不同的金币（不属于 Cpg！）。Cpg 每天可以砍掉其中一棵，并获得其树上所有的金币（怎么会有这种好事）。请你帮助 Cpg 算出他在这 $k$ 天中最多能获得多少金币。

## 输入格式

**本题单测试点内有多组数据**。

每个测试点中有不超过 $10$ 组的测试数据。

每组测试数据：

第一行两个整数 $n,k$。

第二行包含 $n$ 个整数 $m_i$，表示 Cpg 刚看到这 $n$ 棵树时每棵树上的金币数。

第三行 $n$ 个整数 $b_i$，表示每颗摇钱树，每天将会掉落的金币。

以 `0 0` 结束。

## 输出格式

对每组测试数据，输出仅一行，Cpg 在 $k$ 天中能获得的最大金币数。

## 样例

Sample 1:
Input:
3 3
10 20 30
4 5 6
4 3
20 30 40 50
2 7 6 5
0 0
Output:
47
104

## 提示

对于 $100\%$ 的数据，$1 \le n, k \le 10^3$，$1 \le m_i \le 10^5$，$1 \le b_i \le 10^3$。

## 解题思路

### 问题分析
- 有 $n$ 棵摇钱树，每棵树初始有 $m_i$ 个金币，每天掉落 $b_i$ 个金币。
- 可以在 $k$ 天内每天砍一棵树，获得其所有金币。
- 目标是最大化 $k$ 天内获得的总金币。

### 核心思路
- 贪心策略：每天选择当前金币最多的树砍。
- 模拟 $k$ 天的过程，每天找到收益最大的树。

### 算法流程
1. 初始化每棵树的当前金币数为 $m_i$。
2. 对于每天：
   - 找到当前金币最多的树。
   - 砍掉该树，获得其金币。
   - 将其他树的金币增加 $b_i$。
3. 累加每天获得的金币。

### 复杂度分析
- 时间复杂度：$O(k \times n)$，模拟 $k$ 天，每天遍历 $n$ 棵树。
- 空间复杂度：$O(n)$。

## 参考代码

```cpp

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, k;
    while (cin >> n >> k && (n || k)) {
        vector<long long> m(n), b(n);
        for (int i = 0; i < n; i++) cin >> m[i];
        for (int i = 0; i < n; i++) cin >> b[i];
        
        long long ans = 0;
        // 模拟k天，每天选择当前收益最大的树
        vector<long long> cur = m;
        for (int day = 0; day < k; day++) {
            int best = 0;
            for (int i = 1; i < n; i++) {
                if (cur[i] > cur[best]) best = i;
            }
            ans += cur[best];
            cur[best] = -1e18; // 砍掉了
            for (int i = 0; i < n; i++) {
                if (cur[i] > -1e18) cur[i] += b[i];
            }
        }
        cout << ans << "\n";
    }
    return 0;
}

```
