# [RC-05] 排列

## 题目描述
给出 $1\sim n$ 的排列 $p$，下标从 $1$ 开始。

**恰好** $K$ 次任意选择 $1\le i<n$ 并交换 $p_i,p_{i+1}$。问交换完毕后，字典序最小的排列 $p$ 是什么？

## 输入格式
第一行两个非负整数 $n,K$，分别表示排列中数的个数和交换次数。

接下来一行 $n$ 个整数，描述排列 $p$。

## 输出格式
一行以空格隔开的 $n$ 个正整数，描述交换完成之后的排列。

## 样例
### 样例1
输入：
```
5 2
2 1 4 3 5
```
输出：
```
1 2 3 4 5
```
### 样例2
输入：
```
5 3
5 4 3 2 1
```
输出：
```
2 5 4 3 1
```
### 样例3
输入：
```
5 6
5 4 3 2 1
```
输出：
```
1 3 5 4 2
```

## 提示
**本题捆绑测试。**

对于所有数据，$1\le n\le 5\times 10^5$，$0\le K\le 10^{12}$。

详细数据范围如下表：

| Subtask 编号 | $n$ | $K$ | 分数 |
| :-----------: | :-----------: | :-----------: | :-----------: | 
| $1$ | $\le 8$ | |$30$ |
| $2$ | $\le 10^3$ |  |$30$ |
| $3$ | $\le 10^3$ | $=10^{12}$ | $15$ |
| $4$ | $\le 5\times 10^5$ | | $25$ |

## 解题思路

### 问题分析
这是一个关于排列和交换的问题。我们需要在恰好K次相邻交换后，得到字典序最小的排列。

### 核心思路
1. **贪心策略**：为了得到字典序最小的排列，我们应该尽量将小的数字放到前面。
2. **交换次数限制**：每次交换只能交换相邻元素，且恰好交换K次。
3. **关键观察**：每个数字可以向左移动的次数有限，受到K的限制。

### 算法流程
1. 从左到右处理每个位置，对于位置i，我们希望放置当前可用的最小数字。
2. 对于每个数字，计算它移动到当前位置需要的交换次数（即它与当前位置之间的逆序对数量）。
3. 如果剩余交换次数足够，就将该数字移动到当前位置，并减少相应的交换次数。
4. 如果剩余交换次数不够，则选择当前能放置的最小数字。

### 复杂度分析
- 时间复杂度：O(n log n) 或 O(n)，取决于具体实现。
- 空间复杂度：O(n)。
