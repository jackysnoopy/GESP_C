# P2417 [SEERC 2000] 课程

## 思路

核心算法：二分图匹配

## 代码

```cpp
#include <iostream>
#include <vector>
#include <cstring>
using namespace std;

const int MAXN = 20005;

vector<int> graph[MAXN];
int match[MAXN];
bool visited[MAXN];

bool dfs(int u) {
    for (int v : graph[u]) {
        if (!visited[v]) {
            visited[v] = true;
            if (match[v] == -1 || dfs(match[v])) {
                match[v] = u;
                return true;
            }
        }
    }
    return false;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int T;
    cin >> T;
    
    while (T--) {
        int m, n;
        cin >> m >> n;
        
        for (int i = 1; i <= m; i++) {
            graph[i].clear();
        }
        
        for (int i = 1; i <= m; i++) {
            int k;
            cin >> k;
            for (int j = 0; j < k; j++) {
                int p;
                cin >> p;
                graph[i].push_back(p);
            }
        }
        
        memset(match, -1, sizeof(match));
        
        int matched = 0;
        for (int i = 1; i <= m; i++) {
            memset(visited, false, sizeof(visited));
            if (dfs(i)) {
                matched++;
            }
        }
        
        cout << (matched == m ? "YES" : "NO") << "\n";
    }
    
    return 0;
}
```

## 解题思路

### 问题分析
有m个课程和n个可用时间段，每个课程只能安排在特定的时间段内。问是否存在一种安排方案，使得每个课程都有一个专属的时间段（时间段不可重用）。

### 核心思路
将课程和时间段构成二分图：左边m个课程节点，右边n个时间段节点，课程i与时间段j之间连边表示课程i可以安排在时间段j。问题转化为二分图最大匹配：判断最大匹配数是否等于m。使用匈牙利算法（DFS增广路）求解。

### 算法流程
1. 读入每门课程可选的时间段，构建邻接表
2. 初始化match数组，-1表示未匹配
3. 对每门课程调用DFS寻找增广路：
   - 遍历课程的所有可选时间段
   - 若该时间段未访问过，标记已访问
   - 若该时间段未被匹配，或能为当前匹配的课程找到新匹配，则匹配成功
4. 统计成功匹配的课程数
5. 若匹配数等于m，输出"YES"，否则输出"NO"

### 复杂度分析
- 时间复杂度：O(m × (m + n))，匈牙利算法每轮DFS最多O(m+n)
- 空间复杂度：O(m + n)，邻接表和匹配数组
