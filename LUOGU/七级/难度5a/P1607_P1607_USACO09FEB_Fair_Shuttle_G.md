# P1607 [USACO09FEB] Fair Shuttle G

## 题目分析
- 班车经过 `1..N` 站，沿单向运行**一次**，载客容量 `C`。
- `K` 组牛，第 `i` 组：从 `S_i` 上 `E_i` 下，共 `M_i` 头，可以**部分搭乘**。
- 求总搭乘头数最大。

## 解题思路（贪心 + 线段树）
**贪心策略**：按区间右端点 `E_i` 升序处理；对当前组，尽可能多上车（受 `M_i` 与车上余位限制）。
- 直觉：右端点小的组先下车，让位给后面的组；同样的若早下车不上，反而占着更多区间。
- 形式化可证："交换论证"——若一个右端点小的组少坐一头牛而让右端点更大的组多一头，损失 ≥ 收益。

实现需要在区间 `[S_i, E_i-1]`（牛在站 `S..E-1` 之间在车上）查询**当前最大占用**，则可上车人数 = `min(M_i, C - max占用)`。

### 线段树
- 维护 `1..N-1` 站点段（即每段表示该段路途上车里的牛数）的**区间最大值**，支持区间加。
- 每组：
  1. `cur = query([S, E-1])`
  2. `can = min(M, C - cur)`
  3. `ans += can; update([S, E-1], +can)`

### 复杂度
- `O((K+N) log N)`，`K=5e4, N=2e4` 完全可过。

## 实现细节
- 区间使用 `[s, e-1]`：牛在 `s` 上 `e` 下，占据"段"为 `s..e-1`。
- 线段树 `mx[]` 表区间最大值，`tag[]` lazy 加法。
- 注意：取的是**区间最大值**而非平均值——容量限制是逐站不超过 `C`。
- `ans` 用 `long long` 防溢（最大 `5e4·2e4 = 1e9`，理论 int 也行，保险用 ll）。

## 样例验证
样例 `K=8, N=15, C=3`，期望输出 10。算法运行后，按右端点排序处理每组，逐个贪心后输出 `10`，符合预期。

## 复杂度结论
- 时间 `O(K log N + N)`。
- 空间 `O(N)`。
