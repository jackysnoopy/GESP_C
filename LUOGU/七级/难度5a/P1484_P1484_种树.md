# P1484 种树

## 题目描述

cyrcyr 在一条直线上挖了 $n$ 个坑，每个坑可以种树，但相邻两个坑不能同时种。最多种 $k$ 棵树，每个坑种树的获利已知（可以为负）。求最大获利。

## 输入格式

- 第 1 行：两个整数 $n, k$
- 第 2 行：$n$ 个整数，第 $i$ 个数是第 $i$ 个坑种树的获利

## 输出格式

最大获利。

## 数据范围

$1 \le n \le 3\times 10^5$，$1 \le k \le n/2$，$|a_i| \le 10^6$。

## 样例

输入：
```
6 3
100 1 -1 100 1 -1
```
输出：
```
200
```

## 解题思路

### 算法：反悔贪心 + 双向链表 + 大根堆

#### 朴素分析

直接 DP：$f[i][j][0/1]$ 表示前 $i$ 个坑选 $j$ 棵第 $i$ 是否种，复杂度 $O(nk)$，最坏 $n=3\times 10^5,k=1.5\times 10^5$，达到 $4.5\times 10^{10}$，无法接受。

#### 反悔贪心思想

每次贪心地选取当前 `a` 值最大的位置 $p$，加入答案，但需要保证"不相邻"约束。
关键操作：选了 $p$ 后，**禁止** $p$ 的左右邻居 $L,R$ 再被选。但为允许"反悔"——即如果以后发现选 $L,R$ 比选 $p$ 更优——我们引入：

- 将 $p$ 的值替换为 $a[L]+a[R]-a[p]$
- 把 $L,R$ 标记为"已删除"
- 把 $p$（带新值）重新插入堆中

下一轮如果再次选中 $p$，相当于：
1. 撤销之前选 $p$ 的决策（贡献 $-a[p]$）
2. 同时选了 $L$ 和 $R$（贡献 $+a[L]+a[R]$）

如此堆 + 双向链表配合，每次操作 $O(\log n)$，共 $k$ 次，总复杂度 $O((n+k)\log n)$。

### 实现要点

- 用 `priority_queue<pair<long long,int>>` 作大根堆，按 `(值, 位置)` 排序
- 用 `L[],R[]` 维护双向链表
- 删除节点时不一定立即从堆里移除，而是标记 `removed[]`，取堆顶时若不一致则跳过（懒删除）
- 边界：`a[0]=a[n+1]=-INF`，避免反悔时越界

### 复杂度

- 时间：$O((n+k)\log n)$
- 空间：$O(n)$

### 样例验证

样例：`100 1 -1 100 1 -1`，$k=3$。
- 第 1 步选 $a[1]=100$，把 $a[1]$ 改成 $a[0]+a[2]-100 = -INF+1-100$（实际不会再选）。
- 第 2 步选 $a[4]=100$，把 $a[4]$ 改成 $a[3]+a[5]-100=-1+1-100=-100$。
- 第 3 步选剩下的最大值，可能是已经合并的虚拟点；但已无正贡献，停止。
- 答案 $200$ ✓
