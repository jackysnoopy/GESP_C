# P2751 安排活动

## 思路

核心算法：贪心

## 代码

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <queue>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, m1, m2;
    cin >> n >> m1 >> m2;
    
    vector<int> a_times(m1), b_times(m2);
    for (int i = 0; i < m1; i++) {
        cin >> a_times[i];
    }
    for (int i = 0; i < m2; i++) {
        cin >> b_times[i];
    }
    
    // Sort machines by time (fastest first)
    sort(a_times.begin(), a_times.end());
    sort(b_times.begin(), b_times.end());
    
    // Calculate time for A operations
    int time_a = 0;
    priority_queue<int, vector<int>, greater<int>> pq_a;
    for (int t : a_times) {
        pq_a.push(0);
    }
    
    for (int i = 0; i < n; i++) {
        int earliest = pq_a.top();
        pq_a.pop();
        pq_a.push(earliest + a_times[i % m1]);
    }
    
    time_a = pq_a.top();
    
    // Calculate time for B operations
    int time_b = 0;
    priority_queue<int, vector<int>, greater<int>> pq_b;
    for (int t : b_times) {
        pq_b.push(0);
    }
    
    for (int i = 0; i < n; i++) {
        int earliest = pq_b.top();
        pq_b.pop();
        pq_b.push(earliest + b_times[i % m2]);
    }
    
    time_b = pq_b.top();
    
    cout << time_a << " " << time_a + time_b << "\n";
    return 0;
}
```

## 解题思路

### 问题分析
有两类机器需要分别处理n个任务，A类有m1台机器（各自耗时不同），B类有m2台机器（各自耗时不同）。每个任务必须先在A类机器上处理，处理完毕后立即进入B类机器处理。任务在A类和B类之间是串行的。求A类机器完成全部n个任务的时间，以及整个流程（A+B）完成的时间。

### 核心思路
采用贪心策略+优先队列（最小堆）。对于每类机器，每次将任务分配给当前最早空闲的机器。这样可以保证所有机器负载均衡，总完成时间最小。使用最小堆维护每台机器的空闲时间，每次取出堆顶（最早空闲的机器）执行下一个任务。

### 算法流程
1. 读入A类和B类机器的耗时，分别排序
2. 初始化两个最小堆，记录每台机器的空闲时刻（初始为0）
3. 对于每个任务，从堆中取出最早空闲的机器，加上该机器的处理时间后重新入堆
4. A类处理完后，堆顶即为A类完成时间 time_a
5. B类同理处理，堆顶为B类完成时间 time_b
6. 输出 time_a 和 time_a + time_b

### 复杂度分析
- 时间复杂度：O(n log m1 + n log m2)，其中m1、m2为机器数量，n为任务数
- 空间复杂度：O(m1 + m2)，用于存储优先队列
