# [KOI 2024 Round 1] 加倍

## 题目描述

给定一个长度为 $N$ 的正整数序列 $A_1, A_2, \dots, A_N$。希望将该序列变为升序（非降序）排列，即对所有 $1 \le i \le N-1$ 满足 $A_i \le A_{i+1}$。

允许的操作：选择某个下标 $i$，将 $A_i$ 乘以 $2$。可以执行若干次（含零次）。

求将序列排成升序所需的最小操作次数。

## 输入格式

第一行一个整数 $N$。
第二行 $N$ 个整数，表示 $A_1, A_2, \dots, A_N$，以空格分隔。

## 输出格式

输出一个整数，表示将序列变为升序所需的最小操作次数。

## 提示

**样例 1 说明**：对 $A_2$ 和 $A_4$ 各执行两次操作后，序列变为 $[3, 4, 4, 4, 5]$，共 $4$ 次。

**样例 2 说明**：对 $A_2$ 操作两次，$A_4$ 操作三次，$A_5$ 操作一次，最终序列为 $[3, 4, 5, 8, 10]$，共 $6$ 次。

**约束条件**：
- 所有给定的数均为整数。
- $1 \le N \le 250\,000$
- $1 \le A_i \le 1\,000\,000$

时间限制 2000ms，内存限制 1GB。

## 解题思路

### 问题分析

每个元素只能通过不断乘 2 来增大（只能变大不能变小）。要让整个序列非降，必须让每个元素至少不小于它前一个元素最终取到的值。由于操作只能增大数值，且每次操作代价为 1，我们希望每个元素在满足约束的前提下尽可能小，从而给后续元素留出更宽松的空间。

### 核心思路

**贪心**：从左到右依次确定每个元素的最终值。设前一个元素最终值为 `prev`，当前元素原始值为 `cur`。只要 `cur < prev`，就把 `cur` 乘 2 并计数一次，直到 `cur >= prev`。这样当前元素取到了满足非降约束的最小可能值。

**正确性**：操作只能增大数值，因此把当前元素抬得越高，对后续元素只会更不利（后续需要更多次翻倍才能追上）。所以让每个元素"恰好够用"（最小可行值）是全局最优的。任何把当前元素额外抬高的方案都不会减少总操作次数，反而可能增加。这是典型的贪心交换论证。

### 算法流程

1. 读入序列，令 `prev = A[0]`，`ops = 0`。
2. 从 $i = 1$ 到 $N-1$：
   - `cur = A[i]`
   - 当 `cur < prev` 时：`cur *= 2`，`ops++`
   - `prev = cur`
3. 输出 `ops`。

注意用 `long long`，因为反复翻倍后数值会迅速增长（最多约 $60$ 次翻倍即超出 $10^{18}$，仍在 `long long` 范围内）。

### 复杂度分析

- 时间复杂度：每个元素翻倍次数之和。由于 `prev` 每次翻倍至少翻一番，单元素至多约 $60$ 次翻倍，总操作数 $O(N \log V)$，其中 $V$ 为值域上界。$N \le 2.5\times10^5$，完全可以在 2s 内通过。
- 空间复杂度：$O(N)$ 存储序列。

## 参考代码

```cpp
#include <iostream>
#include <vector>
using namespace std;
typedef long long ll;
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n;
    cin >> n;
    vector<ll> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];
    ll prev = a[0], ops = 0;
    for (int i = 1; i < n; i++) {
        ll cur = a[i];
        while (cur < prev) { cur *= 2; ops++; }
        prev = cur;
    }
    cout << ops << "\n";
    return 0;
}
```
