# [CSPro 28] 训练计划

## 题目描述

顿顿要在大赛开幕前 $n$ 天内完成 $m$ 项科目的训练。科目 $i$ 耗时 $t_i$ 天：若从第 $a$ 天开始，则第 $a+t_i-1$ 天为最后一天。

科目间存在依赖：若科目 $i$ 依赖科目 $j$，则只能在 $j$ 训练结束后 $i$ 才能开始（即 $i$ 最早从 $j$ 的结束日次日才开始）。每项科目最多依赖一项别的科目，且依赖科目的编号小于自己；$p_i=0$ 表示无依赖，可从第 1 天开始。多项科目可同一天并行训练。

对每项科目求：
1. 最早开始时间；
2. 在 $n$ 天内完成所有训练的前提下，最晚开始时间。

若无法在 $n$ 天内完成全部训练，则只输出最早开始时间。

## 输入格式

第一行两个正整数 $n, m$（$0 < n \le 365$，$0 < m \le 100$）。

第二行 $m$ 个整数 $p_i$（$0 \le p_i < i$），表示科目 $i$ 依赖的科目编号，$0$ 表示无依赖。

第三行 $m$ 个正整数 $t_i$（$1 \le t_i \le n$），表示科目 $i$ 所需天数。

## 输出格式

第一行输出 $m$ 个正整数，依次为每项科目的最早开始时间。

若可在 $n$ 天内完成全部训练，则第二行输出 $m$ 个正整数，依次为每项科目的最晚开始时间；否则不输出第二行。

## 提示

- $70\%$ 数据满足：无法在 $n$ 天内完成，仅需输出最早开始时间。
- 全部数据：$0 < n \le 365$，$0 < m \le 100$。
- 依赖关系构成森林（每点至多一个父亲，父亲编号更小），按编号递增顺序计算即可。

样例 2 中科目 5 依赖 2、2 依赖 1，故科目 5 最早第 4 天开始；其耗时 10 天，第 13 天结束，超过 10 天，因此无解，不输出最晚开始时间。

## 解题思路

### 问题分析

- 依赖关系是一棵（或若干棵）树，且保证 $p_i < i$，因此按编号从小到大处理时，父亲必先于儿子被处理。
- "最早开始时间"只受依赖约束：一个科目要在其依赖科目结束后才能开始，越早开始越好，故直接沿依赖链向前传递。
- "最晚开始时间"要在保证 $n$ 天内完成、且不违反依赖的前提下尽量推迟。它受两类约束：自身必须在第 $n$ 天前结束；自身必须先于所有依赖自己的科目（儿子）开始。

### 核心思路

- 最早开始时间：`earliest[i] = 1`（无依赖），否则 `earliest[i] = earliest[p_i] + t[p_i]`。按 $i=1\dots m$ 顺序计算。
- 可行性：当且仅当所有科目以最早开始时间出发也能在第 $n$ 天前结束，即 `earliest[i] + t[i] - 1 <= n` 对所有 $i$ 成立。由于科目可并行，这是充要条件。
- 最晚开始时间：从后向前（$i=m\dots 1$）计算，因为儿子编号大于父亲。`latest[i]` 初值取 `n - t[i] + 1`（自身不越界），再对每个依赖 $i$ 的儿子 $j$ 收紧：`latest[i] = min(latest[i], latest[j] - t[i])`（保证 $i$ 在儿子 $j$ 开始前结束）。

### 算法流程

1. 读入 $n,m$、依赖数组 $p$、耗时数组 $t$。
2. 正向计算 `earliest`，输出第一行。
3. 检查可行性；不可行则结束。
4. 建立儿子邻接表（$p_i>0$ 时把 $i$ 加入 $p_i$ 的儿子列表）。
5. 反向计算 `latest`，输出第二行。

### 复杂度分析

- 时间复杂度：$O(m)$。每个科目处理常数次。
- 空间复杂度：$O(m)$，用于存储依赖、耗时、儿子邻接表等。

## 参考代码

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, m;
    cin >> n >> m;
    vector<int> dep(m + 1), t(m + 1);
    for (int i = 1; i <= m; i++) cin >> dep[i];
    for (int i = 1; i <= m; i++) cin >> t[i];
    vector<int> earliest(m + 1), latest(m + 1);
    for (int i = 1; i <= m; i++) {
        if (dep[i] == 0) earliest[i] = 1;
        else earliest[i] = earliest[dep[i]] + t[dep[i]];
    }
    for (int i = 1; i <= m; i++) cout << earliest[i] << (i < m ? " " : "\n");
    bool feasible = true;
    for (int i = 1; i <= m; i++)
        if (earliest[i] + t[i] - 1 > n) feasible = false;
    if (!feasible) return 0;
    vector<vector<int>> children(m + 1);
    for (int i = 1; i <= m; i++)
        if (dep[i] > 0) children[dep[i]].push_back(i);
    for (int i = m; i >= 1; i--) {
        latest[i] = n - t[i] + 1;
        for (int j : children[i])
            latest[i] = min(latest[i], latest[j] - t[i]);
    }
    for (int i = 1; i <= m; i++) cout << latest[i] << (i < m ? " " : "\n");
    return 0;
}
```
