# [蓝桥杯 2025 国 B] 翻倍

## 题目描述

给定 $n$ 个正整数 $A_1, A_2, \ldots, A_n$。每次操作可以选择任意一个数将其翻倍（乘以 $2$）。求让序列单调不下降（每个数都不小于上一个数）所需的最少操作次数。

## 输入格式

第一行一个正整数 $n$。
第二行 $n$ 个正整数 $A_1, A_2, \ldots, A_n$。

## 输出格式

输出一个整数，表示需要的最小操作次数。

## 提示

**样例说明**：可将序列变为 $4, 6, 8, 8, 14, 18$，共 $0+1+2+3+1+1=8$ 次操作。

**评测用例规模与约定**：
- 20%：$n \le 10, A_i \le 100$。
- 50%：$n \le 5000, A_i < 2^{32}$，保证可在所有 $A_i < 2^{32}$ 时满足要求。
- 100%：$1 \le n \le 2\times10^5, 1 \le A_i < 2^{32}$。

时间限制 1000ms，内存限制 256MB。

## 解题思路

### 问题分析

每个数只能通过翻倍变大（不能变小）。要让序列非降，必须让每个数至少不小于前一个数最终取到的值。由于操作只能增大数值且每次代价为 1，应让每个数在满足约束的前提下**尽可能小**，从而给后续留出更宽松的空间。

### 核心思路

**贪心**：从左到右依次确定每个数的最终值。设前一个数最终值为 `prev`，当前数为 `cur`。只要 `cur < prev`，就把 `cur` 翻倍并计数，直到 `cur >= prev`。这样当前数取到满足非降约束的最小可能值。

**正确性**：操作只能增大数值，把当前数抬得越高对后续越不利（后续需要更多次翻倍才能追上）。因此让每个数"恰好够用"是全局最优的，任何额外抬高都不会减少总操作次数，反而可能增加。这是典型的贪心交换论证。

### 算法流程

1. 读入序列，令 `prev = A[0]`，`ops = 0`。
2. 从 $i = 1$ 到 $n-1$：
   - `cur = A[i]`
   - 当 `cur < prev` 时：`cur *= 2`，`ops++`
   - `prev = cur`
3. 输出 `ops`。

由于 $A_i < 2^{32}$，反复翻倍后数值会迅速增长（最多约 $60$ 余次翻倍即超出 $2^{63}$），必须用 `long long` 防止溢出。

### 复杂度分析

- 时间复杂度：每个数翻倍次数之和。`prev` 每次翻倍至少翻一番，单元素至多约 $60$ 次翻倍，总操作数 $O(n \log V)$，$n \le 2\times10^5$，可在 1s 内通过。
- 空间复杂度：$O(n)$。

## 参考代码

```cpp
#include <iostream>
#include <vector>
using namespace std;
typedef long long ll;
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n;
    cin >> n;
    vector<ll> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];
    ll prev = a[0], ops = 0;
    for (int i = 1; i < n; i++) {
        ll cur = a[i];
        while (cur < prev) { cur *= 2; ops++; }
        prev = cur;
    }
    cout << ops << "\n";
    return 0;
}
```
