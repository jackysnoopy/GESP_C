# 星降る海

## 题目描述

![](https://cdn.luogu.com.cn/upload/image_hosting/q3hd0pkn.png)

给定一个长为 $n$ 的颜色序列 $a$ 和一个正整数 $k$。你可以执行以下操作任意次（可以为 $0$ 次）。
  - 选择一个 $\{1,2,\ldots n\}$ 的子集 $S$ 和颜色 $c$ 并将 $S$ 中所有位置的颜色变成 $c$。

求至少需要几次操作使得极长颜色段个数**恰好**等于 $k$，并构造一组解。

如果存在多组合法的解，你可以输出任意一组。

## 输入格式

第一行两个正整数 $n,k$。

第二行 $n$ 个正整数 $a_i$，表示颜色序列。

## 输出格式

第一行输出最小操作次数 $t$。

接下来 $t$ 行，按顺序输出每次操作的方案。每行第一个整数 $s_i$ 表示你的集合大小，接下来 $s_i$ 个互不相同的 $1$ 到 $n$ 的整数表示你选择的集合，最后输出一个 $1$ 到 $n$ 的整数 $c$ 表示更改的颜色。

## 样例

### 样例输入 1
```
5 3
1 2 3 3 3
```

### 样例输出 1
```
0
```

### 样例输入 2
```

1 2 3 3 3
```

### 样例输出 2
```

1 2 1
```

### 样例输入 3
```

```

### 样例输出 3
```

```

## 解题思路

### 问题分析
给定长度为 n 的颜色序列 a，每次操作可以选择任意子集 S 和颜色 c，将 S 中所有位置变为颜色 c。目标是用最少的操作使极长颜色段（连续相同颜色的最长段）个数恰好等于 k，并构造方案。

首先统计当前序列的极长颜色段个数 m（合并相邻相同颜色后计算）。注意一次操作可以同时修改多个不相邻的位置。

### 核心思路
- 若 m = k：不需要操作，输出 0
- 若 m > k：需要减少颜色段。每次操作可以将某些段的边界元素变为相邻段的颜色，从而合并段。具体地，每次操作可以选择若干个段边界位置，将其颜色改为相邻段的颜色，每次操作最多减少若干段。贪心策略是每次操作尽可能多地合并段。
- 若 m < k：需要增加颜色段。每次操作可以将某个段中的部分元素改为新颜色，从而拆分段。每次操作最多增加若干段。

关键观察：一次操作可以同时修改任意多个位置。因此：
- 减少段数时：每次操作可以选取多个段边界，将边界元素改为与相邻段同色，从而合并。每次操作最多可以减少的段数取决于可操作的边界数量。
- 增加段数时：每次操作可以从某个颜色段中选取部分位置改为新颜色，每次操作可以增加的段数取决于段的长度。

### 算法流程
1. 读入 n, k 和颜色序列
2. 统计当前极长颜色段个数 m
3. 若 m == k，输出 0 并结束
4. 若 m > k：
   - 需要减少 m - k 个段
   - 每次操作选择若干段边界位置，将其改为相邻段颜色
   - 操作次数 = ⌈(m - k) / 每次可减少的最大段数⌉，具体构造需贪心选择边界
5. 若 m < k：
   - 需要增加 k - m 个段
   - 每次操作从较长的段中选取部分位置改为新颜色来拆分段
   - 操作次数 = ⌈(k - m) / 每次可增加的最大段数⌉
6. 输出操作次数及每次操作的具体方案

### 复杂度分析
- 时间复杂度：O(n)，统计段数 O(n)，构造方案 O(n)
- 空间复杂度：O(n)，用于存储序列和操作方案
