# 排除干扰

## 题目描述
小 R 和小 M 各有 $m$ 张牌，牌有 $n$ 类。两人轮流弃牌（R 先手），每人弃到只剩一张时，R 得分 $a_{u,v}$（$u,v$ 为剩余牌类型），M 得分 $-a_{u,v}$。交互博弈，$c=0$ 扮演 R，$c=1$ 扮演 M，需达到最优得分。

## 输入格式
第一行 $n, m, c$。接下来 $n \times n$ 矩阵 $a$。然后两行分别为 R 和 M 的初始牌分布。

## 输出格式
交互式：轮到你时输出弃牌类型，轮到对手时读入其弃牌。游戏结束时读入 0 或 -1。

## 样例
见题目描述。

## 提示
$1 \le n \le 10^3$，$1 \le m \le 10^4$。

## 解题思路
### 问题分析
游戏最终状态是每人剩一张牌。由于双方都能看到对方手牌并最优决策，这本质上是一个矩阵博弈。

### 核心思路
R 选择保留类型 $i$ 使 $\min_j a_{i,j}$ 最大（maximin），M 选择保留类型 $j$ 使 $\max_i a_{i,j}$ 最小（minimax）。游戏值为 maximin = minimax。交互中只需保留最优类型，弃掉其余牌。

### 算法流程
1. 计算最优保留类型
2. 轮到自己时弃掉非保留类型的牌
3. 轮到对手时读入其弃牌

### 复杂度分析
时间 $O(n^2 + m)$，空间 $O(n^2)$。
