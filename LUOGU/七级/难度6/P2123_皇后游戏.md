# P2123 皇后游戏

## 思路

核心算法：贪心排序

## 代码

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

struct Minister {
    long long a, b;
};

bool compare(const Minister& x, const Minister& y) {
    if (x.a * x.b != y.a * y.b) {
        return x.a * x.b < y.a * y.b;
    }
    return x.a < y.a;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int T;
    cin >> T;
    
    while (T--) {
        int n;
        cin >> n;
        
        vector<Minister> ministers(n);
        for (int i = 0; i < n; i++) {
            cin >> ministers[i].a >> ministers[i].b;
        }
        
        // Sort by a * b
        sort(ministers.begin(), ministers.end(), compare);
        
        // Calculate minimum maximum reward
        long long sum_a = 0;
        long long prev_reward = 0;
        long long max_reward = 0;
        
        for (int i = 0; i < n; i++) {
            sum_a += ministers[i].a;
            long long reward = max(prev_reward, sum_a) + ministers[i].b;
            max_reward = max(max_reward, reward);
            prev_reward = reward;
        }
        
        cout << max_reward << "\n";
    }
    
    return 0;
}
```

## 解题思路

### 问题分析
有n个大臣，每个大臣有两个属性(a, b)。大臣按顺序从左到右排列，每个大臣会获得奖励 = max(之前所有大臣a的和, 上一个大臣的奖励) + 当前大臣的b。目标是通过调整大臣排列顺序，使得最后一个大臣获得的奖励最小。

### 核心思路
贪心排序。每个大臣的贡献体现在两方面：其a值会累加给后面所有大臣的基数，其b值会直接加到奖励中。最优排序策略是按a×b的乘积从小到大排列（乘积相同时按a排序）。这是因为对于相邻两个大臣x和y，交换它们的位置会使奖励变化为：y的b值对x之后的人产生影响，比较a×b可以判断交换是否更优。

### 算法流程
1. 读入n个大臣的(a, b)值
2. 按a×b从小到大排序（乘积相同时按a排序）
3. 按顺序遍历大臣：
   - 累加sum_a（之前所有大臣a的和）
   - 计算当前大臣的奖励 = max(prev_reward, sum_a) + b
   - 更新max_reward
4. 输出max_reward

### 复杂度分析
- 时间复杂度：O(n log n)，排序占主导
- 空间复杂度：O(n)，存储大臣数组
