# P3462 [POI 2007] ODW-Weights

## 题目描述

Byteotian实验物理研究所在搬迁过程中遇到了一个后勤问题——转移其大量精密砝码的任务变得非同寻常。

研究所拥有若干个强度有限的容器。需要尽可能多地将砝码放入容器中，剩余的将被丢弃。除了不超过容器强度的要求外，对放入容器的砝码数量没有限制。容器也可以为空。

任意两个砝码具有一个特殊的性质：其中一个的质量是另一个的整数倍。特别地，它们可能有相同的质量。

任务：编写一个程序，读取容器的强度和砝码的质量，确定最多可以放入容器的砝码数量。

## 输入格式

第一行包含两个整数 $n$ 和 $m$（$1 \le n, m \le 100\,000$），分别表示容器数量和砝码数量。

第二行包含 $n$ 个整数 $w_i$（$1 \le w_i \le 1\,000\,000\,000$），表示容器的强度。

第三行包含 $m$ 个整数 $m_j$（$1 \le m_j \le 1\,000\,000\,000$），表示砝码的质量。

## 输出格式

一行，包含一个整数——最多可以放入容器的砝码数量。

## 样例

输入：
```
2 4
13 9
4 12 2 4
```
输出：
```
3
```

## 解题思路

### 问题分析

所有砝码质量满足整除关系，排序后可以用贪心策略。

### 核心思路

1. 将所有砝码按质量排序（相同质量的放在一起）
2. 从大到小枚举每个砝码质量作为"基准质量"
3. 对于每个基准质量，贪心地将能放入的砝码放入当前最强的容器

### 算法流程

1. 排序砝码，按质量从大到小分组
2. 从大到小尝试每组砝码，用贪心分配到容器中
3. 统计最大放入数量

### 复杂度分析

- 时间复杂度：$O(n \log n + m \log m)$
- 空间复杂度：$O(n + m)$
