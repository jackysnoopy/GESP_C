# P6787 [SWTR-6] Snow Mountain

## 题目描述

有 $n$ 个水晶（$n$ 为偶数），第 $i$ 个水晶的能量值为 $a_i$，且所有 $a_i$ 互不相同。

每次操作可以选择两个未被摧毁的水晶一起摧毁，释放的能量为 $\min(a_i, a_j) \times k$，其中 $k$ 是当前是第几次操作（从 $1$ 开始）。

存在一些限制：对于某些水晶 $i$，有 $x_i$ 表示水晶 $i$ 和水晶 $x_i$ 不能在同一轮一起被摧毁（$x_i = -1$ 表示没有限制，注意下标从 $1$ 开始）。

求摧毁所有水晶的最小总能量，以及摧毁方案。如果无解输出 $-1$。

## 输入格式

第一行一个正整数 $n$，表示水晶数量。

第二行 $n$ 个正整数 $a_i$，表示能量值。

第三行 $n$ 个整数 $x_i$，表示限制。

## 输出格式

如果无解，输出一行 $-1$。

否则第一行输出最小总能量，接下来 $\frac{n}{2}$ 行，每行两个数 $u,v$ 表示第几轮摧毁的水晶编号（从 $1$ 编号）。

## 样例

### 样例输入 1

```
4
1 4 2 3
3 -1 -1 2
```

### 样例输出 1

```
4
2 3
1 4
```

### 样例输入 3

```
4
1 9 4 5
4 -1 4 2
```

### 样例输出 3

```
-1
```

## 提示

- $2 \le n \le 5 \times 10^5$
- $1 \le a_i \le 10^9$，所有 $a_i$ 互不相同
- $x_i \in \{-1, 1, 2, \ldots, n\}$，$x_i \ne i$

## 解题思路

### 问题分析

每轮选一对水晶，释放较小值乘以轮次的能量。要最小化总能量，应该让较小的水晶在较早的轮次被摧毁。

### 核心思路

贪心策略：按能量值从大到小处理。对于当前最大的未摧毁水晶，尽量将它与最小的可配对水晶配对。这样较大的水晶不会在后期被"浪费"在早期轮次中。

具体来说：
1. 将水晶按能量值从大到小排序
2. 维护一个可用水晶的集合（从小到大排列）
3. 对于当前最大的水晶，找到最小的可配对水晶进行配对
4. 如果找不到可配对的水晶，输出 $-1$

### 算法流程

1. 读入数据，按能量值排序
2. 从大到小处理每个水晶
3. 对于当前水晶 $u$，在可用集合中从最小的开始尝试配对
4. 检查 $x_i$ 限制，找到第一个可配对的 $v$
5. 记录配对，累加能量，标记两个水晶已使用
6. 如果某轮找不到配对，输出 $-1$

### 复杂度分析

- 时间：$O(n^2)$（暴力扫描），可通过更优的数据结构优化到 $O(n \log n)$
- 空间：$O(n)$
