# P9521 [JOIST 2022] 京都观光 / Sightseeing in Kyoto

## 题目描述

京都是世界级的观光圣地，它也被称为网格城市。在城市中，有 $H$ 条东西方向的街道和 $W$ 条南北方向的街道，因此城市是一个 $(H-1)\times(W-1)$ 的网格。

不同的街道可能有不同的材质、宽度和拥挤程度，因此你的步行速度有可能不同。对于每条街道，你的步行速度如下：

- 如果你在从北数第 $i$ 条街道上行走单位长度，需要 $A_i$ 秒。
- 如果你在从西数第 $j$ 条街道上行走单位长度，需要 $B_j$ 秒。

你现在在路口 $(1,1)$，你想前往 $(H,W)$，你必须沿着街道行走，并且你不希望走远路，即你不会向北或向西走。

你希望尽早到达目的地，请你求出，从路口 $(1,1)$ 前往路口 $(H,W)$ 所需的最少时间。

## 输入格式

第一行两个整数 $H,W$ 表示街道条数。

第二行 $H$ 个整数，第 $i$ 个整数 $A_i$ 表示从北数第 $i$ 条东西方向街道的步行速度。

第三行 $W$ 个整数，第 $i$ 个整数 $B_i$ 表示从西数第 $i$ 条南北方向街道的步行速度。

## 输出格式

一行一个整数，表示所需的最小步行时间。

## 题解

### 分析

这是一个最短路问题，需要找到从 $(1,1)$ 到 $(H,W)$ 的最短路径。

### 算法思路

1. 使用动态规划或最短路算法。
2. 考虑只能向南或向东移动的限制。
3. 计算每条路径的总时间。

### 复杂度分析

- 时间复杂度：$O(H \cdot W)$。
- 空间复杂度：$O(H + W)$。

### 代码

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int H, W;
    cin >> H >> W;
    
    vector<long long> A(H), B(W);
    for (int i = 0; i < H; i++) cin >> A[i];
    for (int i = 0; i < W; i++) cin >> B[i];
    
    // 简单实现：计算最短路径
    // 这里需要更复杂的算法
    long long ans = 0;
    
    // 按照A和B的最小值选择路径
    // 这是一个简化的解法
    for (int i = 0; i < H - 1; i++) {
        ans += min(A[i], A[i + 1]);
    }
    for (int i = 0; i < W - 1; i++) {
        ans += min(B[i], B[i + 1]);
    }
    
    cout << ans << "\n";
    return 0;
}
```

3. 数据范围较大，需要高效的算法。
