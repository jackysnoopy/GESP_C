# [JSOI2010] 缓存交换

## 题目描述
CPU只能和Cache直接交换数据。给定N次主存访问和Cache容量M，每次Cache缺失时选择换出哪个单元以最小化总缺失次数。

## 输入格式
第一行N和M，第二行N个正整数（主存块编号）。

## 输出格式
一行，Cache缺失次数的最小值。

## 样例
输入：
```
6 2
1 2 3 1 2 3
```
输出：
```
4
```

## 提示
N ≤ 10^5, M ≤ N。

## 解题思路

### 问题分析
经典的最优离线缓存问题。LRU不是最优的，最优策略是Belady算法。

### 核心思路
Belady算法：当需要换出时，选择下次访问最远的那个块换出（或永远不会被再次访问的块）。

### 算法流程
1. 预处理每个位置的下一次出现位置
2. 使用优先队列维护Cache中各块的下次访问位置
3. Cache命中时更新下次访问位置
4. Cache缺失且满时，换出下次访问最远的块

### 复杂度分析
- 时间：O(N log N)
- 空间：O(N)
