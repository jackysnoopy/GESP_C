# P6902 [ICPC 2014 WF] Surveillance

## 题目描述

一个凸多边形建筑有 n 条边。你有 k 个摄像头，每个摄像头放置在某条边上，可以覆盖若干条相邻的边。求最少需要多少个摄像头才能覆盖所有边，或者输出 impossible。

## 输入格式

第一行两个整数 n, k。接下来 k 行，每行两个整数 a_i, b_i，表示第 i 个摄像头从边 a_i 覆盖到边 b_i（沿逆时针方向）。

## 输出格式

最少摄像头数或 impossible。

## 数据范围

- 3 ≤ n ≤ 10^6
- 1 ≤ k ≤ 10^6

## 样例

**样例1：**
```
100 7
...
输出: 3
```

**样例2：**
```
8 2
...
输出: impossible
```

**样例3：**
```
8 2
...
输出: 2
```

## 题解

### 核心思路

环形覆盖问题。将凸多边形的边视为环上的点，摄像头覆盖连续的一段区间。

**断环为链**：枚举起点，将环展开为长度 2n 的链，用贪心求最小覆盖。

### 算法步骤

1. 枚举从每条边开始作为起点
2. 对每个起点，贪心选择能覆盖到最远位置的摄像头
3. 检查是否能在 k 个以内覆盖所有边
4. 取所有起点的最小值

### 优化

用倍长和单调队列优化，将复杂度从 O(nk) 降到接近线性。

### 复杂度

- 时间：O(n + k log k)（预处理）+ O(n)（枚举起点）
- 空间：O(n)
