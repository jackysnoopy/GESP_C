# P6000 [CEOI2016] match

## 题目描述

给定一个字符串 $s$，构造一个字典序最小的合法括号序列，使得该括号序列与 $s$ 匹配。

**匹配定义**：两个序列长度相等，且对于每一对匹配的括号（左括号位置 $i$ 和右括号位置 $j$），满足 $s_i = s_j$。

如果没有合法的括号序列，输出 $-1$。

## 输入格式

一行一个字符串 $s$。

## 输出格式

一行一个括号序列或 $-1$。

## 样例

### 样例输入

```
abbaaa
```

### 样例输出

```
(()())
```

## 数据范围

- $2 \le n \le 10^5$
- 字符串 $s$ 仅包含小写英文字母

## 算法思路

### 贪心策略

我们需要构造字典序最小的合法括号序列。由于左括号 `(` 的字典序小于右括号 `)`，我们应该尽可能早地放置左括号，但必须保证最终能够形成合法序列。

关键观察：每一对匹配的括号必须对应字符串中相同的字符。因此，对于每种字符，左括号和右括号的数量必须相等。

### 算法步骤

1. 统计每种字符的总出现次数 `total[c]`。
2. 从左到右扫描字符串，维护每种字符已放置的左括号数量 `left[c]`。
3. 对于每个位置 $i$：
   - 如果 `left[s[i]] < total[s[i]] / 2`，说明还可以放置左括号，则放置 `(` 并将位置入栈。
   - 否则，必须放置右括号 `)`。此时需要检查栈是否非空且栈顶字符与当前字符相同（否则无法匹配），然后弹出栈顶。
4. 扫描结束后，如果栈非空，说明存在未匹配的左括号，输出 $-1$；否则输出构造的括号序列。

### 正确性证明

- **合法性**：栈保证了括号的嵌套结构。每次放置右括号时，都与栈顶的左括号匹配，且字符相同。
- **字典序最小**：我们优先放置左括号，只有当无法再放置左括号时（即 `left[c]` 已达到上限）才放置右括号。
- **完备性**：如果存在合法解，该算法一定能找到；如果算法输出 $-1$，则无解。

## 复杂度分析

- 时间复杂度：$O(n)$，其中 $n$ 是字符串长度。我们只遍历一次字符串，栈操作均为 $O(1)$。
- 空间复杂度：$O(n)$，用于存储栈和结果字符串。

## 代码实现

```cpp
#include <iostream>
#include <string>
#include <vector>
#include <stack>
#include <cstring>

using namespace std;

const int MAXN = 100005;

int total[256];
int left_cnt[256];

int main() {
    string s;
    cin >> s;
    int n = s.size();
    
    memset(total, 0, sizeof(total));
    for (char c : s) {
        total[(int)c]++;
    }
    
    memset(left_cnt, 0, sizeof(left_cnt));
    stack<int> st;
    string ans;
    
    for (int i = 0; i < n; ++i) {
        char c = s[i];
        int idx = (int)c;
        if (left_cnt[idx] < total[idx] / 2) {
            ans += '(';
            left_cnt[idx]++;
            st.push(i);
        } else {
            if (st.empty() || s[st.top()] != c) {
                cout << -1 << endl;
                return 0;
            }
            ans += ')';
            st.pop();
        }
    }
    
    if (!st.empty()) {
        cout << -1 << endl;
    } else {
        cout << ans << endl;
    }
    
    return 0;
}
```

## 注意事项

1. 使用 `memset` 初始化数组，注意头文件 `<cstring>`。
2. 字符作为数组下标时，需要转换为 `int` 类型。
3. 栈中存储的是位置索引，方便检查栈顶字符。
4. 当 `total[c]` 为奇数时，`total[c] / 2` 会向下取整，此时 `left[c]` 最多只能达到 `total[c] / 2`，无法完全匹配，但算法会自然处理这种情况。
