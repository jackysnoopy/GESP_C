# [EER1] 代价

## 题目描述
给出一个长度为 $n+2$ 的序列 $a_i$，其中第 $1$ 个数和第 $n+2$ 个数固定为 $1$。你每次可以选择序列中间的一个数删除（不能是第一个和最后一个），删除位置 $p$ 上的数的代价为 $a_{p-1} \times a_p \times a_{p+1}$。你需要执行这个操作直到无法操作为止。求最小的代价和。

## 输入格式
第一行一个正整数 $n$。

第二行 $n$ 个正整数，第 $i$ 个数表示 $a_{i+1}$。

## 输出格式
一行一个正整数，表示最小的代价和。

## 样例
### 样例 1
```
3
1 2 3
```
```
9
```

### 样例 2
```
4
19 26 8 17
```
```
846
```

### 样例 3
```
6
1 1 1 1 1 1
```
```
6
```

## 解题思路

### 问题分析
给定 n 和 k，求 1..n 的排列中恰好有 k 个逆序对的排列数。

### 核心思路
使用动态规划，dp[i][j] 表示 1..i 的排列中恰好有 j 个逆序对的方案数。关键观察：将数字 i 插入到 i-1 个数的排列中，可以产生 0 到 i-1 个新的逆序对。

### 算法流程
1. 初始化 dp[0][0] = 1（空排列有 0 个逆序对）
2. 对每个 i 从 2 到 n：
   - 计算前缀和 ps[j] = dp[0] + dp[1] + ... + dp[j]
   - 对每个 j，dp[i][j] = ps[j] - ps[max(0, j-(i-1))-1]
3. 输出 dp[n][k]

### 复杂度分析
- 时间复杂度：O(n*k)，使用前缀和优化
- 空间复杂度：O(k)，使用滚动数组
