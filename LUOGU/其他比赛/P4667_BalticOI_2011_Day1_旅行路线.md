# [BalticOI 2011 Day1] 旅行路线

## 题目描述
欧拉路径存在性与构造。

## 输入格式
无向图。

## 输出格式
路径或无解。

## 样例
### 样例 1
```
5 5
1 2
2 3
3 4
4 5
1 5
```
```
1 2 3 4 5
```

## 解题思路
### 问题分析
本题要求在无向图中找到一条经过所有边恰好一次的路径（欧拉路径/Euler Trail）。首先需要判断欧拉路径是否存在，若存在则构造该路径。

### 核心思路
1. **欧拉路径存在性判定**：无向图存在欧拉路径当且仅当图是连通的（忽略孤立点），且度数为奇数的节点恰好有 0 个或 2 个。若奇度数节点为 0 个，则为欧拉回路；若为 2 个，则必须从其中一个奇度数节点出发。
2. **构造方法（Hierholzer 算法）**：从起点出发，用 DFS 沿着未访问的边前进，当某个节点无法再前进时将其加入路径尾部。最终得到的反向路径即为欧拉路径。使用栈模拟递归实现，避免系统栈溢出。

### 算法流程
1. 读入边，构建邻接表，统计每个节点的度数
2. 判断连通性（BFS/DFS），并统计奇度数节点个数
3. 若奇度数节点不为 0 或 2，输出无解
4. 若奇度数节点为 2，从其中一个出发；否则从任意有边的节点出发
5. 运行 Hierholzer 算法构造欧拉路径
6. 输出路径（注意 Hierholzer 输出是反序，需逆序）

### 复杂度分析
- 时间复杂度：$O(V + E)$，其中 $V$ 为节点数，$E$ 为边数
- 空间复杂度：$O(V + E)$
