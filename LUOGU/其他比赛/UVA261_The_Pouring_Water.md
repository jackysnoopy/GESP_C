# UVA261 - The Pouring Water（倒水问题）

## 题目描述

给定两个容量分别为 a 和 b 的水壶（a ≤ b），通过以下六种操作，判断是否能量出恰好 c 升水：

1. 装满水壶 a
2. 装满水壶 b
3. 倒空水壶 a
4. 倒空水壶 b
5. 从水壶 a 倒水到水壶 b（直到 a 空或 b 满）
6. 从水壶 b 倒水到水壶 a（直到 b 空或 a 满）

## 输入格式

多组测试数据，每行三个整数 a、b、c（1 ≤ a ≤ b ≤ 100，0 ≤ c ≤ 100），以 EOF 结束。

## 输出格式

对每组输入，输出一行 `yes` 或 `no`，表示是否能量出恰好 c 升水。

## 样例输入

```
3 5 4
3 5 1
3 5 7
```

## 样例输出

```
yes
yes
no
```

## 解题思路

### 核心思想

本题是经典的**裴蜀定理（Bézout's identity）**应用。

### 数学原理

两个水壶 a 和 b 通过倒水操作能量出的水量，恰好是 gcd(a, b) 的倍数（且不超过 b）。

**定理**：能量出恰好 c 升水，当且仅当以下条件之一成立：
- c = 0（不需要水）
- 0 < c ≤ b 且 c % gcd(a, b) == 0

**证明简述**：
- 裴蜀定理告诉我们，存在整数 x、y 使得 ax + by = gcd(a, b)
- 因此，通过倒水操作可以量出 gcd(a, b) 的任意整数倍（不超过 b）
- 由于 b 本身就是 b 的倍数且 b % gcd(a, b) == 0，所以 b 总是可以量出

### 边界情况

- c > b：超出大壶容量，不可能量出
- c = 0：不需要水，直接输出 yes
- a = b：只能量出 0 和 b

## 时间复杂度

- 时间复杂度：$O(\log(\min(a, b)))$，即 GCD 的计算时间
- 空间复杂度：$O(1)$
