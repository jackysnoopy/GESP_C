# UVA166 - Change Making (找零问题)

## 题目描述

给定6种面额的硬币（5, 10, 20, 50, 100, 200便士），每种有若干枚。给定一个需要找零的金额，求用最少的硬币数凑出该金额。

## 输入格式

每组数据包含7行：
- 前6行：每种面额硬币的数量（可能有0枚）
- 第7行：需要找零的金额（单位：元）
- 以 `0 0 0 0 0 0` 结束

## 输出格式

最少需要的硬币数量

## 样例输入

```
5 5 5 5 5 5
1.00
0 0 0 0 0 0
```

## 样例输出

```
5
```

## 解题思路

这是一道多重背包问题。使用动态规划：
- `dp[i]` 表示凑出金额 `i` 所需的最少硬币数
- 对每种面额，从大到小枚举使用数量
- 答案为 `dp[target]`

注意金额需要转换为整数（乘以100或200）以避免浮点数精度问题。

## 复杂度

- 时间复杂度：O(6 × target × max_coins)
- 空间复杂度：O(target)
