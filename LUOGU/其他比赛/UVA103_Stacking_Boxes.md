# UVA103 - Stacking Boxes

## 题目描述

给定 n 个 k 维盒子，每个盒子有 k 个维度的尺寸。确定最长的嵌套链，使得链中每个盒子都可以嵌套在下一个盒子中。

**嵌套条件**：盒子 A 可以嵌套在盒子 B 中，当且仅当将 A 和 B 的维度分别从小到大排序后，A 的每个维度都严格小于 B 的对应维度。

## 输入格式

多组测试数据，以文件结束符（EOF）结尾。每组数据：
- 第一行包含两个整数 k 和 n，分别表示维度数和盒子数量（1 ≤ k ≤ 30, 1 ≤ n ≤ 30）
- 接下来 n 行，每行包含 k+1 个正整数：第一个整数是盒子编号（标签），后面 k 个整数是该盒子的 k 个维度

## 输出格式

对于每组测试数据，输出两行：
- 第一行：最长嵌套链的长度
- 第二行：嵌套链中盒子的编号（从小到大排列，即从最小的盒子到最大的盒子，用空格分隔）

## 样例输入

```
2 5
3 4 3
5 3 5
6 4 6
2 6 3
1 1 5
```

## 样例输出

```
2
1 2
```

## 解题思路

### 核心思想

这是一个经典的**最长递增子序列（LIS）**问题在多维空间的推广，可以使用**动态规划**求解。

### 算法步骤

1. **预处理**：对每个盒子的 k 个维度进行升序排序。排序后，判断盒子 A 能否嵌套在盒子 B 中，只需依次比较排序后的维度，检查是否全部严格小于。

2. **排序盒子**：将所有盒子按维度字典序排序（先比较第一维，若相同则比较第二维，以此类推）。排序后，若盒子 A 能嵌套在盒子 B 中，则 A 一定排在 B 之前。这保证了 DP 的无后效性。

3. **动态规划**：
   - `dp[i]`：以第 i 个盒子结尾的最长嵌套链长度
   - 状态转移：`dp[i] = max(dp[j] + 1)`，其中 j < i 且盒子 j 能嵌套在盒子 i 中
   - 初始化：`dp[i] = 1`（每个盒子自身构成长度为 1 的链）

4. **路径重建**：
   - 用 `prev[i]` 记录以 i 结尾的最长链中 i 的前驱
   - 找到 `dp` 最大值对应的终点，沿 `prev` 回溯重建路径

5. **输出**：先输出最长链长度，再输出路径中的盒子编号（从小到大）。

### 嵌套判断

将每个盒子的维度排序后，盒子 A 能嵌套在盒子 B 中的条件为：对所有维度 d，`A.dims[d] < B.dims[d]`（严格小于）。

### 样例分析

输入中 k=2, n=5，5 个盒子的排序后维度为：
- 盒子 1: [1, 5]
- 盒子 2: [3, 6]
- 盒子 3: [3, 4]
- 盒子 5: [3, 5]
- 盒子 6: [4, 6]

嵌套关系（A → B 表示 A 可嵌套在 B 中）：
- 1→2: 1<3 ✓, 5<6 ✓
- 1→6: 1<4 ✓, 5<6 ✓
- 3→6: 3<4 ✓, 4<6 ✓
- 5→6: 3<4 ✓, 5<6 ✓

最长链长度为 2（例如 1→2），输出 `2` 和 `1 2`。

## 复杂度分析

- **时间复杂度**：O(n² × k)
  - 对每个盒子的维度排序：O(n × k × log k)
  - 对盒子排序：O(n × k × log n)
  - 动态规划（双层循环 + 嵌套判断）：O(n² × k)
  - 总体：**O(n² × k)**

- **空间复杂度**：O(n × k)
  - 存储盒子信息：O(n × k)
  - dp 和 prev 数组：O(n)
  - 总体：**O(n × k)**

## 代码实现要点

1. 每个盒子的维度先排序再比较，这是嵌套判断的关键
2. 盒子整体按维度字典序排序，保证 DP 的正确性
3. `prev` 数组记录路径前驱，用于回溯重建嵌套链
4. 处理多组测试数据，直到 EOF
