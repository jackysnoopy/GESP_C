# UVA100 - The 3n+1 problem

## 题目描述

对于给定的正整数 $n$，按以下规则生成序列：

- 如果 $n$ 是偶数，则 $n = n / 2$
- 如果 $n$ 是奇数，则 $n = 3n + 1$

重复此过程直到 $n = 1$。定义**循环长度**为从初始值到 $n=1$ 所需的步数（包含 $n=1$ 本身）。

对于给定的区间 $[i, j]$，求该区间内所有整数的最大循环长度。

**注意**：输入中 $i$ 可能大于 $j$，需先比较大小后取闭区间。

## 输入格式

多组测试数据，每行两个整数 $i$ 和 $j$，以 EOF 结束。

## 输出格式

对每组输入，输出一行：

```
i j 最大循环长度
```

其中 $i$、$j$ 为原始输入顺序。

## 样例输入

```
1 10
100 200
201 210
900 1000
```

## 样例输出

```
1 10 20
100 200 125
201 210 89
900 1000 174
```

## 解题思路

1. **直接模拟**：对区间内每个数按规则计算循环长度。
2. **记忆化搜索**：用数组缓存已计算的结果，避免重复计算。当计算过程中遇到已缓存的值时直接返回，大幅减少运算量。
3. **long long 中间变量**：奇数 $3n+1$ 可能超过 $10^6$，但后续会快速下降回已缓存范围。

## 时间复杂度

- 空间复杂度：$O(M)$，其中 $M = 10^6$ 为记忆化数组大小。
- 时间复杂度：摊销 $O(N \cdot \text{avg\_steps})$，其中 $N = j - i + 1$，记忆化使得总体计算量远小于朴素方法。

## 空间复杂度

$O(M)$，用于存储记忆化数组。
