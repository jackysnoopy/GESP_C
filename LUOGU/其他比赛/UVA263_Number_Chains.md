# UVA263 - Number Chains (数字链)

## 题目描述

对于给定的正整数，按以下规则生成序列：

1. 将数字的各位按**降序**排列，得到最大排列
2. 将数字的各位按**升序**排列，得到最小排列
3. 计算差值：最大排列 - 最小排列

重复此过程，直到差值在之前出现过（形成循环）。输出每次变换的结果、链的长度（即变换次数）以及最后一个唯一的数。

**注意**：多组测试数据之间用空行分隔。输入以 `0` 结束。

## 输入格式

每组数据一行，包含一个正整数（最多 9 位）。以 `0` 结束输入。

## 输出格式

对每组输入：
- 第一行：`原始数字是 N`
- 接下来若干行：每次变换的结果
- 最后一行：`链长为 L`

## 样例输入

```
1234
0
```

## 样例输出

```
原始数字是 1234
3087
8352
6174
3087
链长为 3
```

## 解题思路

1. **字符串模拟**：将数字转为字符串，用 `sort` 分别按升序和降序排列，得到最小和最大排列。
2. **集合去重**：用 `set<long long>` 记录已出现过的数，判断何时形成循环。
3. **循环终止条件**：当新产生的差值已在集合中出现时，停止并输出链长。

## 时间复杂度

- 每步排序：$O(d \log d)$，其中 $d$ 为数字位数（最多 9）。
- 集合查找：$O(\log L)$，$L$ 为链长。
- 总体：$O(L \cdot d \log d)$，链长 $L$ 很小（通常 < 20）。

## 空间复杂度

$O(L)$，用于存储集合中的已见数字。
