# UVA308 - Radar (雷达)

## 题目大意

在 x 轴上放置雷达，每个雷达的覆盖半径为 R。平面上有 n 个目标点，每个点需要被至少一个雷达覆盖。求最少需要多少个雷达。

输入多组数据，每组包含目标数量 n 和雷达半径 R，以及各目标点的坐标。

## 算法思路

**贪心 + 区间调度**：

1. **转换问题**：对于每个目标点 (x, y)，如果 y > R 则无法覆盖。否则，雷达可以放在 x 轴上的区间 `[x - d, x + d]` 内，其中 `d = sqrt(R² - y²)`。
2. **区间调度**：问题转化为：在 x 轴上选最少的点，使得每个区间至少包含一个点。
3. **贪心策略**：按区间右端点排序，每次在当前未覆盖区间的最左右端点处放置雷达。

## 正确性证明

按右端点排序后贪心选择，是经典的区间覆盖问题的最优解。每次选择能覆盖当前区间且位置最靠右的点（即右端点），可以最大化对后续区间的覆盖能力。

## 复杂度

- 时间：O(n log n)，排序主导
- 空间：O(n)

## 关键点

- 问题转化：将二维覆盖问题降为一维区间覆盖
- 经典贪心策略：按右端点排序 + 贪心放置
- 注意处理 y > R 的不可达情况
