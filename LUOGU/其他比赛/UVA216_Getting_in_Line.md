# UVA216 - Getting in Line (接入线路)

## 题目描述

有 n 台计算机分布在平面上，需要用电缆依次连接它们（形成一条路径）。每根电缆需要额外加 16 英尺（用于连接两端的设备）。要求找到使总电缆长度最小的连接顺序。

## 输入格式

多组测试数据。每组第一行 n，接下来 n 行每行两个整数 x, y 表示计算机坐标。n=0 结束。

## 输出格式

输出连接顺序和每段电缆长度，以及总长度。

## 解题思路

### 问题分析
寻找经过所有点的最短路径（类似旅行商问题）。

### 核心思路
- 由于 n ≤ 8，可以用全排列暴力搜索
- 枚举所有排列，计算总距离（欧氏距离 + 16）
- 找到最小值

### 算法流程
1. 计算所有点对之间的欧氏距离
2. 枚举全排列，累加相邻点距离 + 16
3. 记录最优排列

### 复杂度分析
- 时间复杂度：O(N! × N)
- 空间复杂度：O(N²)
