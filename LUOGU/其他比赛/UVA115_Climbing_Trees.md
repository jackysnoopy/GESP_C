# UVA115 - Climbing Trees

## 题目描述

给定一组父子关系描述一棵树，然后回答关于祖先关系的查询。查询格式为两个节点名，需要判断它们之间的关系：

- 如果一个节点是另一个的直接子节点，输出 "child of ..."
- 如果一个节点是另一个的祖先（隔代），输出 "descendant of ..."
- 如果它们在同一层，输出 "sibling" 或 "cousin"（不同父节点）
- 如果没有关系，输出 "no relation"

## 输入格式

- 父子关系列表（每行两个名字，直到 "no.child" 标记结束）
- 查询列表（每行两个名字）

## 输出格式

- 对每个查询输出对应的关系描述

## 解题思路

### 核心思想

使用**哈希表**存储父子关系，然后通过**向上追溯**判断两个节点之间的关系。

### 算法步骤

1. 读入父子关系，建立 parent 映射
2. 对每个查询，分别向上追溯两个节点
3. 检查是否在某层相遇，判断关系类型

### 关键点

- 使用 map 存储 parent 关系
- 向上追溯时记录深度
- 需要处理各种关系类型

## 复杂度分析

- **时间复杂度**：O(n × q)，n 为节点数，q 为查询数
- **空间复杂度**：O(n)
