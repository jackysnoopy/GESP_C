# UVA104 - Arbitrage

## 题目描述

套利是指利用货币汇率的差异，通过一系列兑换操作，使得最终获得的货币数量多于起始数量。

给定n种货币之间的汇率矩阵，其中`rate[i][j]`表示1单位货币i可以兑换`rate[i][j]`单位货币j。请你判断是否存在套利机会，即是否存在一种货币兑换序列，使得从某种货币出发，经过一系列兑换后回到该货币时，获得的金额大于起始金额。

如果存在套利机会，输出兑换序列；否则输出"no arbitrage sequence exists"。

## 输入格式

- 多组测试数据，每组数据以整数n开头（1 ≤ n ≤ 20）
- 接下来是n×n的汇率矩阵，第i行第j列表示货币i到货币j的汇率
- 对角线元素（i=j）为1.0
- 输入以EOF结束

## 输出格式

- 如果存在套利机会，输出兑换序列（包含起点和终点，起点和终点相同）
- 如果不存在套利机会，输出"no arbitrage sequence exists"

## 样例输入

```
3
1.0 2.0 3.0
0.5 1.0 0.5
0.33 2.0 1.0
3
1.0 2.0 3.0
0.5 1.0 1.0
0.33 2.0 1.0
0
```

## 样例输出

```
1 2 3 1
no arbitrage sequence exists
```

## 解题思路

### 核心思想

本题可以使用**修改版Floyd-Warshall算法**求解。传统的Floyd-Warshall用于求最短路径（加法），而本题需要求最大汇率乘积（乘法）。

### 算法步骤

1. **初始化**：
   - `rate[i][j]`表示从货币i到货币j的最佳汇率
   - `next_node[i][j]`表示从i到j路径中，i的下一个节点

2. **Floyd-Warshall迭代**：
   - 对于每个中间货币k，检查是否通过k可以获得更好的汇率
   - 更新条件：`rate[i][k] * rate[k][j] > rate[i][j]`
   - 同时更新路径数组`next_node`

3. **检查套利**：
   - 检查对角线元素`rate[i][i]`是否大于1.01（允许1%的利润空间）
   - 如果存在这样的i，则说明有套利机会

4. **路径重建**：
   - 从i开始，沿着`next_node`数组回溯直到回到i
   - 输出完整的兑换序列

### 关键点

- 为什么阈值是1.01而不是1.0？由于浮点数精度问题，直接判断>1.0可能产生误判，因此使用1.01作为阈值
- 路径重建使用`next_node`数组，类似于最短路径中的前驱数组

## 复杂度分析

- **时间复杂度**：O(n³)，其中n是货币种类数
  - Floyd-Warshall三层循环：O(n³)
  - 检查套利和路径重建：O(n²)
  
- **空间复杂度**：O(n²)
  - 需要存储汇率矩阵和路径数组
