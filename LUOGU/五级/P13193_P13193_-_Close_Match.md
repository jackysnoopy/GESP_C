# P13193 - Close Match

## 题目描述

你正在观看一场重要的体育比赛。记分牌上显示着两个得分（可能有前导零），但由于球的撞击，部分数字损坏了，显示为 `?`。

你需要用数字替换所有的 `?`，使得两个得分的绝对差最小。如果有多种方案差值相同，则选择使第一个得分（Coders）最小的方案；如果仍有多种，则选择使第二个得分（Jammers）最小的方案。

## 输入输出格式

**输入**：第一行 T，接下来 T 行，每行两个等长字符串 C 和 J，由数字和 `?` 组成。

**输出**：`Case #x: c j`，其中 c 和 j 是替换 `?` 后的字符串。

## 样例

| 输入 | 输出 |
|------|------|
| `1? 2?` | `Case #1: 19 20` |
| `?2? ??3` | `Case #2: 023 023` |
| `? ?` | `Case #3: 0 0` |
| `?5 ?0` | `Case #4: 05 00` |

## 解题思路

字符串长度最多18位，数字范围可达 $10^{18}$，无法暴力枚举。需要**分类讨论**策略。

### 核心思想：枚举分界点

两个字符串 C 和 J 长度相同，考虑从左到右逐位比较：

1. **C == J 的情况**：差为0，最优。如果能在所有位置填出相同数字，直接取该方案。
2. **在某个位置 pos 处分叉**：pos 之前 C 和 J 完全相同，在 pos 处一个比另一个大，之后各自取极值。

对于位置 pos 处分叉的情况：

- **C[pos] > J[pos]**：pos 之后 C 尽量小（填 `'0'`），J 尽量大（填 `'9'`）。
- **C[pos] < J[pos]**：pos 之后 C 尽量大（填 `'9'`），J 尽量小（填 `'0'`）。

对于每个 pos，枚举满足条件的最小差值对 `(cp, jp)` 即可。

### 三重比较优先级

- 第一优先：绝对差最小
- 第二优先：C 的值最小
- 第三优先：J 的值最小

## 算法原理

1. 先尝试 C == J（差=0 的最优情况）
2. 枚举每个分界位置 pos（0 到 n-1）
3. 对于每个 pos，先确保前面所有位相同
4. 在 pos 位枚举所有合法的 `C[pos] > J[pos]` 或 `C[pos] < J[pos]` 组合
5. 用三重比较选出全局最优

## 代码逐段注释

```cpp
struct Ans {
    ll diff;      // 绝对差值
    string c, j;  // 填充后的两个字符串
    // 三重比较：差值 < C值 < J值
    bool operator<(const Ans& o) const {
        if (diff != o.diff) return diff < o.diff;
        if (c != o.c) return c < o.c;
        return j < o.j;
    }
};
```

每个分界位置只尝试差值最小的组合（找到第一组就 break），因为更大的差值不可能更优。

## 复杂度分析

- **时间**：$O(T \times n \times 100)$，n 最多 18，非常高效
- **空间**：$O(n)$
